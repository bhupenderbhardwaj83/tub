History & Foundations

React Native (2015) revolutionized mobile dev by allowing React developers to build native apps.

  • The "Bridge" Era: JSON serialization between JS and Native threads. Slow but effective.
  • New Architecture (Fabric/TurboModules): JSI (C++) interface allowing synchronous communication.
  • Expo: The "Rails" for React Native. Now the standard entry point.
02

Underlying Architecture

Hermes Engine

An optimized JavaScript engine built by Facebook specifically for React Native. Reduces TTI (Time to Interaction) and APK size.

Fabric Renderer

The new UI system. Moves logic from the "Bridge" to C++ core, allowing finer priority control for gestures and animations.

JSI

JavaScript Interface. Allows JS to call C++ (Native) functions directly by reference, removing the JSON serialization bottleneck.

State Management

Use Zustand for client state (Sidebar, Theme, Auth Token) and TanStack Query for Server State.

Note: Avoid Redux unless you have complex transaction logic. Zustand is 90% smaller and boilerplate-free.

09

Performance

  • 01

    Use FlatList over ScrollView

    FlatList virtualizes elements (only renders what is on screen). ScrollView renders everything at once.

  • 02

    Memoize Components

    React.memo() and useCallback() prevent unnecessary re-renders in large lists.

  • 03

    Image Caching

    Use expo-image instead of the default Image component for aggressive caching and improved performance.

Auth & Security

FastAPI Backend Integration

Secure the app using JWTs.

  1. Login: Send credentials to Python FastAPI (/token).
  2. Store: Save Access/Refresh tokens in expo-secure-store (NOT AsyncStorage).
  3. Interceptor: Axios interceptor attaches Bearer token to every request.
  4. Refresh: On 401, mostly silently refresh token using refresh token.

Practical Examples

1. Login Screen with FastAPI

A complete login form using React Hook Form and Zod validation.

const LoginScreen = () => {
  const { control, handleSubmit } = useForm({
    resolver: zodResolver(loginSchema)
  });
  const login = useAuthStore(s => s.login);

  const onSubmit = async (data) => {
    try {
      await login(data.email, data.password);
    } catch (err) {
      Alert.alert('Error', 'Invalid credentials');
    }
  };

  return (
    <SafeAreaView style={tw`flex-1 bg-white justify-center p-4`}>
       <Controller
         control={control}
         name="email"
         render={({ field: { onChange, value } }) => (
           <TextInput 
             style={tw`border p-4 rounded mb-4`} 
             placeholder="Email"
             value={value}
             onChangeText={onChange}
           />
         )}
       />
       <TouchableOpacity 
         onPress={handleSubmit(onSubmit)}
         style={tw`bg-blue-600 p-4 rounded items-center`}
       >
         <Text style={tw`text-white font-bold`}>Sign In</Text>
       </TouchableOpacity>
    </SafeAreaView>
  );
};

2. Authenticated Dashboard

Fetching secure data passing the Bearer token.

// api/client.ts
client.interceptors.request.use(async (config) => {
  const token = await SecureStore.getItemAsync('token');
  if (token) config.headers.Authorization = `Bearer ${token}`;
  return config;
});

// screens/Dashboard.tsx
const { data } = useQuery({ 
  queryKey: ['dashboard'], 
  queryFn: () => client.get('/dashboard/stats').then(r => r.data) 
});
