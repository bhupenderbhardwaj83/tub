The Modern Edge Router

Traefik Reverse Proxy

The Cloud Native Application Proxy. Stop managing messy Nginx configs and start routing with automatic discovery, dynamic configuration, and production-grade reliability.

🤔

What is Traefik?

Traefik (pronounced traffic) is a modern HTTP reverse proxy and load balancer that makes deploying microservices easy. It integrates with your existing infrastructure components (Docker, Swarm, Kubernetes, AWS, etc.) and configures itself automatically and dynamically.

🔄 Reverse Proxy vs Forward Proxy

  • ✓ Forward Proxy: Sits in front of a client (you). Example: A corporate VPN that hides your IP from the internet.
  • ✓ Reverse Proxy (Traefik): Sits in front of servers. It takes incoming requests from the internet and routes them to the correct internal service (container, VM, function).

🚀 Why Traefik Exists?

Traditional proxies (Nginx, HAProxy) usually require a static config file reload every time you add a service. In the age of containers where services spin up/down in seconds, this doesn't scale.

Traefik listens to your container orchestrator (like Docker) and updates its routing rules in real-time, instantly.

Core Architecture

    INTERNET
       │
       ▼
┌────────────── EntryPoints (Ports) ──────────────┐
│  :80 (HTTP)           :443 (HTTPS)              │
└──────┬──────────────────────┬───────────────────┘
       │                      │
       ▼                      ▼
┌──────────────── Routers (Rules) ────────────────┐
│  Host(`api.io`)       Host(`web.io`)            │
│  Path(`/v1`)          PathPrefix(`/app`)        │
└──────┬──────────────────────┬───────────────────┘
       │                      │
       ▼                      ▼
┌────────────── Middlewares (Tweaks) ─────────────┐
│  BasicAuth            Compress                  │
│  RateLimit            Headers                   │
└──────┬──────────────────────┬───────────────────┘
       │                      │
       ▼                      ▼
┌────────────── Services (Balancer) ──────────────┐
│  Load Balancer (Round Robin, Weighted)          │
└──────┬──────────────────────┬───────────────────┘
       │                      │
       ▼                      ▼
┌────────────────── Providers ────────────────────┐
│  Docker Container 1    Docker Container 2       │
└─────────────────────────────────────────────────┘

1. EntryPoints

The network ports Traefik listens on (usually 80 & 443). The doorways into your cluster.

2. Routers

The logic. Analyzes incoming requests (Host, Path, Headers) and connects them to a service.

3. Middlewares

The modifiers. Sit between Router and Service. Tweaks requests (Add headers, Auth, Compress).

4. Services

The abstract destination. Configures how to reach the actual backend (Load balancing).

Installation Methods

docker-compose.yml (Recommended) 
version: "3.8"

services:
  traefik:
    image: traefik:v3.0
    container_name: traefik
    command:
      # Enable Docker Provider (read labels)
      - --providers.docker=true
      # Do not expose containers by default
      - --providers.docker.exposedbydefault=false
      # Define EntryPoints
      - --entrypoints.web.address=:80
      - --entrypoints.websecure.address=:443
    ports:
      - "80:80"
      - "443:443"
      # Dashboard port (only valid if api.insecure=true, not recommended for prod)
      - "8080:8080"
    volumes:
      # Traefik needs to talk to the Docker Socket
      - /var/run/docker.sock:/var/run/docker.sock:ro
      # Store Certificates persistently
      - ./letsencrypt:/letsencrypt
    networks:
      - proxy

networks:
  proxy:
    external: true
PRO TIP: Network Isolation

Always create an external Docker network (e.g., `proxy`) and connect only the containers that need public access to it.
Run: docker network create proxy

Routing Deep Dive

Host Based Routing

Route based on the domain name.

Host(`example.com`) || Host(`www.example.com`)

Path Based Routing

Route based on URL path prefix.

PathPrefix(`/api`)

Priority Rules

Traefik calculates route length. Longer = Higher Priority.

Path(`/api/v2`) > Path(`/api`)
You can manually set specific priority in labels if needed.

Header Based

Route based on custom headers (great for A/B testing).

Headers(`X-Beta-User`, `true`)
🔒

TLS & HTTPS (ACME)

Traefik can automatically generate and renew SSL certificates via Let's Encrypt using the ACME protocol. No more cron jobs or certbot scripts.

Configuration in `traefik.yml` or `command`

# 1. Define the Resolver
certificatesResolvers:
  myresolver:
    acme:
      email: you@example.com
      storage: /letsencrypt/acme.json
      # Challenge Type: HTTP-01 (Easiest, requires port 80 open)
      httpChallenge:
        entryPoint: web

# 2. Force HTTPS Redirect (Global)
entryPoints:
  web:
    address: ":80"
    http:
      redirections:
        entryPoint:
          to: websecure
          scheme: https

HTTP-01 Challenge

Traefik puts a file on a specific route. Let's Encrypt calls your domain on Port 80 to verify.

Best for simple setups

DNS-01 Challenge

Traefik adds a TXT record to your DNS provider API. Confirms ownership without port 80/443 pointing to this server.

Required for Wildcards (*.domain.com)

Traefik with Docker Labels

This is where the magic happens. You don't touch Traefik config to add a service. You just label the service container.

whoami-service.yml 
services:
  whoami:
    image: traefik/whoami
    labels:
      # 1. Enable Traefik for this container
      - "traefik.enable=true"
      
      # 2. Define the Router Rule (Domain)
      - "traefik.http.routers.whoami.rule=Host(`whoami.localhost`)"
      
      # 3. Use the HTTPS EntryPoint
      - "traefik.http.routers.whoami.entrypoints=websecure"
      
      # 4. Use the Cert Resolver named 'myresolver'
      - "traefik.http.routers.whoami.tls.certresolver=myresolver"
      
      # 5. Connect to middleware (e.g., BasicAuth)
      - "traefik.http.routers.whoami.middlewares=my-auth"

Middlewares Deep Dive

Authentication

Protect any route with Basic Auth easily.

# Define Middleware
labels:
  - "traefik.http.middlewares.my-auth.basicauth.users=admin:$$apr1$$KEY...Hash"
  
# Attach to Router
  - "traefik.http.routers.my-app.middlewares=my-auth"

StripPrefix

Route /api/v1/users -> backend /users

labels:
  - "traefik.http.middlewares.my-strip.stripprefix.prefixes=/api/v1"

Rate Limiting

Prevent DoS attacks.

labels:
  - "traefik.http.middlewares.limit.ratelimit.average=100"
  - "traefik.http.middlewares.limit.ratelimit.burst=50"

⛓️ Chaining Middlewares

Combine multiple middlewares into a chain.

labels:
  - "traefik.http.middlewares.secured.chain.middlewares=my-auth,limit"

Traefik Dashboard

Traefik comes with a built-in dashboard to visualize the state of your routes. By default in dev mode it's on port 8080. In production, always secure it.

⚠️

Security Warning

NEVER expose the dashboard insecurely to the public internet. It exposes infrastructure details.

Secure Method: 
labels:
  - "traefik.enable=true"
  - "traefik.http.routers.dashboard.rule=Host(`traefik.example.com`)"
  - "traefik.http.routers.dashboard.service=api@internal"
  - "traefik.http.routers.dashboard.middlewares=auth" # MANDATORY

Advanced Use Cases

Canary Releases

Gradually shift traffic from v1 to v2.

# Weighted Load Balancing (Traefik Service Config)
traefik.http.services.app-v1.loadbalancer.server.port=80
traefik.http.services.app-v2.loadbalancer.server.port=80

# The Weighted Service
traefik.http.services.app-canary.weighted.services.app-v1.weight=80
traefik.http.services.app-canary.weighted.services.app-v2.weight=20

Internal Tools (VPN Mode)

Use Middleware IPAllowList to restrict access to internal tools (like Grafana, Portainer) only to your VPN IP.

"traefik.http.middlewares.vpn-only.ipallowlist.sourcerange=10.0.0.0/24, 192.168.1.1"
Master Class Scenario

Single Server, Multiple Apps, One Proxy

The "Holy Grail" of single-server orchestration. How to run a centralized Traefik instance that handles traffic for App A (e.g., WordPress) and App B (e.g., NodeJS API), where each app lives in its own isolated `docker-compose.yml` file.

                            │
                            ▼
          ┌───────────────────────────────────┐
          │      HOST: myserver.com           │
          │                                   │
          │  ┌─────────────────────────────┐  │    ┌ uses ┐
          │  │      TRAEFIK PROXY          │◄─┼────┤ Docker │
          │  │     (ports 80/443)          │  │    │ Socket │

          │        ▲                 ▲        │
          │        │                 │        │
          └────────┼─────────────────┼────────┘
                   │                 │
     ┌─────────────┴──┐           ┌──┴─────────────┐

     │ └─────┬──────┘ │           │ └────────────┘ │
     │       │        │           │                │

Traffic flows through the shared `proxy-net` while databases remain isolated in private networks.
1

Create Global Network

This external network acts as the bridge. All public containers must plugin to this switch.

docker network create proxy-net
2

Central Traefik Compose

Does not know about apps yet. Just listens on the network.

/opt/traefik/docker-compose.yml
services:
  traefik:
    image: traefik:v3.0
    command:
      - --providers.docker=true
      # Watch containers on other networks too? No, usually not needed if they join this one.
      - --providers.docker.exposedbydefault=false
    ports:
      - "80:80"
      - "443:443"
    networks:
      - proxy-net   # <--- Connects to the Bridge
    volumes:
      - /var/run/docker.sock:/var/run/docker.sock:ro

networks:
  proxy-net:
    external: true  # <--- Use existing network
3

Application A (e.g. Blog)

Connects to `proxy-net` ONLY for routing. Uses `internal-net` for DB.

/home/user/blog/docker-compose.yml
services:
  wordpress:
    image: wordpress
    labels:
      - "traefik.enable=true"
      - "traefik.http.routers.blog.rule=Host(`blog.example.com`)"
      - "traefik.docker.network=proxy-net"  # <--- CRITICAL: Tell Traefik which IP to use
    networks:
      - proxy-net  # Public side
      - default    # Private side (Talks to DB)

  db:
    image: mysql
    networks:
      - default    # Private only. Traefik can't reach this.

networks:
  proxy-net:
    external: true
4

Application B (e.g. API)

Completely separate folder/project. Same logic.

/home/user/api/docker-compose.yml
services:
  api:
    image: my-node-app
    labels:
      - "traefik.enable=true"
      - "traefik.http.routers.api.rule=Host(`api.example.com`)"
      - "traefik.docker.network=proxy-net"
    networks:
      - proxy-net

networks:
  proxy-net:
    external: true
💡
Common Pitfall: Gateway Timeout?

If you get a 504 Gateway Timeout, it often means Traefik CANNOT reach the container.
Ensure you set traefik.docker.network=proxy-net label on the app container if it is attached to multiple networks.

Security Best Practices

  • ✓
    Use Strict TLS: Set minimum TLS version to v1.2 or v1.3 in Traefik static config.
  • ✓
    Secure Headers: Use `headers` middleware to add HSTS, X-Frame-Options, and CSP.
  • ✓
    Socket Security: Never expose `docker.sock` to the internet. If you can, use a socket proxy or run Traefik as non-root (more complex).
  • ✓
    Fail2Ban: Integrate with Fail2Ban by parsing Traefik access logs to ban bad actors.

Troubleshooting

Usually means the rule didn't match. Check:
  • Is `exposedbydefault` set to false? Did you add `traefik.enable=true`?
  • Is the Host header correct in your request?
  • Check the Dashboard to see if the Router exists.
  • Check logs! Traefik logs ACME errors clearly.
  • "Casper" staging errors? You might have hit rate limits.
  • Ensure Port 80 is open to the world for HTTP-01 challenge.

Traefik is not just a config file; it's a dynamic router. It asks Docker "Who are you?" and configures the path to them instantly.

EntryPoints open the door.
Routers decide where to go.
Middlewares dress up the request.

Top 50 Reference Commands

A quick-reference guide to the most essential configuration keys, labels, and CLI arguments. Grouped by core concept.

🐳 Docker & Discovery

Command / Label / Key Core Concept & Usage
--providers.docker=true Enables the Docker provider. Tells Traefik to listen to Docker socket events.
--providers.docker.exposedbydefault=false Security best practice. Prevents exposing every container automatically. (Configure in static file/CLI).
traefik.enable=true Label: Explicitly tells Traefik to route traffic to this container. Required if exposedbydefault=false.
traefik.docker.network=name Label: Specifies which Docker network IP Traefik should use to reach the container. Crucial for multi-network setups.
--api.insecure=true Enables the dashboard on port 8080 without auth. DEV ONLY.
--log.level=DEBUG Sets logging verbosity. Essential for troubleshooting "Why isn't my route working?".

🛣️ Routers & Rules

Label Key Core Concept & Usage
traefik.http.routers.NAME.rule The main entry point. Defines WHEN this router is active.
Host(`example.com`) Matches the exact domain name in the request header.
HostRegexp(`{sub}.domain.com`) Matches domains using regex pattern matching.
Path(`/api`) Matches exact path. `example.com/api` matches, `example.com/api/foo` does NOT.
PathPrefix(`/api`) Matches path start. `example.com/api/foo` matches. Most common for APIs.
Headers(`Key`, `Value`) Matches specific request headers. Great for API versioning or internal routing.
traefik.http.routers.NAME.entrypoints Assigns router to specific ports (e.g., `websecure` for 443).
traefik.http.routers.NAME.priority Manual priority override (Integer). Higher number = Higher priority.

🔒 TLS & Certificates

Command / Label Core Concept & Usage
traefik.http.routers.NAME.tls=true Enables TLS (HTTPS) on the router.
traefik.http.routers.NAME.tls.certresolver=lev Tells Traefik which resolver (defined in static config) to use for getting the cert.
--certificatesresolvers.lev.acme.email Configures the email for Let's Encrypt notifications (Static Config).
--certificatesresolvers.lev.acme.storage Path to `acme.json` to store certificates. Must be persisted via volume.
...acme.httpchallenge.entrypoint=web Use HTTP-01 challenge. Must use port 80.
...acme.dnschallenge.provider=cloudflare Use DNS-01 challenge. Required for wildcard certs.

⚖️ Services (Load Balancing)

Label Key Core Concept & Usage
traefik.http.services.NAME.loadbalancer.server.port Tells Traefik which internal container port to send traffic to (e.g., 3000, 8080).
...loadbalancer.sticky.cookie=true Enables Sticky Sessions. User stays on the same container.
...loadbalancer.passhostheader=true Forwards the original `Host` header to the backend application (default true).

🔧 Middlewares

Label Key / Middleware Type Core Concept & Usage
traefik.http.routers.NAME.middlewares=list Attaches one or more middlewares (comma separated) to a router.
...middlewares.NAME.basicauth.users BasicAuth: Protects route with user/password (htpasswd format).
...middlewares.NAME.stripprefix.prefixes StripPrefix: Removes path prefix before sending to backend (/api/v1/user -> /user).
...middlewares.NAME.redirectscheme.scheme RedirectScheme: Force redirect from HTTP -> HTTPS.
...middlewares.NAME.headers.customrequestheaders Headers: Inject custom headers into the request (e.g., X-Forwarded-For).
...middlewares.NAME.compress=true Compress: Gzip compresses the response body.
