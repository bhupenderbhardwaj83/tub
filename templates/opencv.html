Introduction

OpenCV (Open Source Computer Vision Library) is an open-source computer vision and machine learning software library. It enables machines to extract meaning from images and videos, providing fast, low-level building blocks for real-time applications.

Origins & Creators

Officially launched in 1999, OpenCV was an Intel initiative to advance CPU-intensive applications. The primary creator was Gary Bradski, and the first major release appeared in 2000. Vadim Pisarevsky managed the Russian team. It was built to provide a common infrastructure for computer vision apps.

Community & Future

Supported by the non-profit OpenCV.org and a massive developer community. The future focuses heavily on Deep Learning (DNN) and Edge AI, optimizing models for mobile/embedded devices like Raspberry Pi and NVIDIA Jetson.

Industry Adoption

Used by giants like Google, Yahoo, Microsoft, Intel, IBM, Sony, Honda, Toyota. From stitching street views to surveillance, monitoring mine equipment, robot navigation, and product inspection.

Core Concepts

In OpenCV, images are represented as NumPy arrays. Each pixel is numerical data.

What is OpenCV

💡 What Is It?

OpenSource Computer Vision Library. A huge open-source library for computer vision, machine learning, and image processing.

🔧 Why It Exists?

To provide a common infrastructure for computer vision applications and to accelerate the use of machine perception in commercial products.

⚙️ How It Works?

Written in C++ for performance, with wrappers for Python. It processes images as NumPy arrays (matrices of pixel values).

🎯 Where Used?
  • Facial Recognition
  • Automated Inspection
  • Robotics & Navigation
  • Medical Image Analysis
  • Security Surveillance
✅ When To Use?

For real-time image processing operations, edge detection, feature extraction, and when you need efficient C++ backed performance.

⚠️ When To Avoid?

Training massive Deep Learning models from scratch (use PyTorch/TensorFlow). Complex GUI applications (use Qt/Kivy instead of HighGUI).

Image I/O

What Is It?

Fundamental functions to load images from disk to memory, display them on screen, and save processed results back to disk.

How It Works?

Decodes file formats (JPG, PNG, TIFF) into dense NumPy arrays (Matrix of BGR pixels) and encodes arrays back to files.

Where Used?

The starting and ending point of almost every computer vision pipeline (Input → Process → Output).

When To Use?

When reading datasets, capturing frames, or debugging pipelines by visualizing intermediate steps.

When To Avoid?

Avoid `imshow` on headless servers (Docker/Cloud) as it requires a display (GUI). Use `imwrite` instead.

Function Description Inputs Returns Module Example

cv2.imshow Displays image in window window, image None highgui cv2.imshow("img", img)

Critical: Window Management

OpenCV windows require cv2.waitKey() to process events. Without it, the window will freeze or close immediately.

cv2.waitKey(0)
Wait forever (Key press needed)
Usage: Image Viewers
cv2.waitKey(1)
Wait 1 ms (Real-time)
Usage: Video Processing

Common Mistakes ❌

  • Forgetting waitKey → Window freezes
  • Using waitKey(0) in video loop → Video stops at first frame
  • Forgetting destroyAllWindows → Zombie windows remain open

Filtering & Blurring

What Is It?

Convolution operations that modify pixel values based on their neighbors.

How It Works?

Slides a kernel (small matrix) over the image, calculating a new value (e.g., average) for the center pixel.

Where Used?

Denoising (removing grain), smoothing, and preparing images for edge detection.

When To Use?

When images are noisy or too sharp/detailed for the intended analysis algorithm.

When To Avoid?

Heavy blurring destroys fine details (edges/textures) critical for recognition tasks.

Thresholding

What Is It?

The simplest method of image segmentation: converting grayscale to binary (black/white).

How It Works?

Compares each pixel to a threshold value. If > val, sets to max (255), else min (0).

Where Used?

Separating foreground objects (text, tools) from background.

When To Use?

When there is high contrast between object and background (e.g., scanned documents).

When To Avoid?

Global thresholding fails with varying lighting. Use Adaptive Thresholding instead.

Edge Detection

What Is It?

Process of identifying points in a digital image at which the image brightness changes sharply.

How It Works?

Computes the gradient (derivative) of image intensity. Large gradients = Edges.

Where Used?

Lane detection, fingerprint recognition, and structure analysis.

When To Use?

To extract the structural layout of an object while discarding color/texture info.

When To Avoid?

Directly on noisy images. Always apply Gaussian Blur before Edge Detection.

Curve joining all continuous points (along the boundary), having same color or intensity.

How It Works?

Morphological Operations

What Is It?

Operations based on image shape using a structuring element (kernel).

How It Works?

Probes the image with a small shape template (kernel) to erode or dilate boundaries.

Where Used?

Removing noise (Opening), filling holes (Closing), separating joined objects.

When To Use?

To clean up binary masks created by simple thresholding.

When To Avoid?

Large kernels can destroy small, important object features.

Function Description Inputs Returns Module Example

cv2.morphologyEx (OPEN) Erosion followed by dilation image, op, kernel image imgproc cv2.morphologyEx(th,cv2.MORPH_OPEN,kernel)
cv2.morphologyEx (CLOSE) Dilation followed by erosion image, op, kernel image imgproc cv2.morphologyEx(th,cv2.MORPH_CLOSE,kernel)

Geometric Transformations

What Is It?

Modifying the spatial arrangement of pixels (rotating, shifting, warping).

How It Works?

Maps input coordinates to output coordinates using matrix multiplication (Affine/Perspective).

Where Used?

Document scanning (rectifying perspective), image alignment, data augmentation.

When To Use?

When the camera angle is skewed or objects need to be aligned to a template.

When To Avoid?

Extreme warping causes pixelation. Requires interpolation (e.g., Cubic/Linear).

Feature Detection & Matching

What Is It?

Finding "interesting" points (corners, blobs) that are unique and recognizable.

How It Works?

Analyzes local pixel patterns to be invariant to scale, rotation, and brightness (e.g., ORB, SIFT).

Where Used?

Image stitching (Panoramas), 3D reconstruction, Object tracking.

When To Use?

When matching objects across different views or occlusions.

When To Avoid?

On texture-less surfaces (plain walls) where no unique keypoints exist.

Object Detection (Classical)

What Is It?

Locating specific objects (Faces, Eyes, Cars) within an image.

How It Works?

Uses machine learning classifiers (Haar Cascades, HOG) trained on pos/neg samples.

Where Used?

Face unlock, license plate recognition, pedestrian detection.

When To Use?

For detecting rigid, well-defined objects like frontal faces or eyes (fast & light).

When To Avoid?

For complex objects, occlusions, or varying orientations. Using Deep Learning (YOLO/SSD) instead is recommended for modern accuracy.

What Is It?

Processing a sequence of images (frames) over time.

What Is It?

Technique to separate moving foreground objects from a static background.

How It Works?

Builds a model of the background over time and compares current frame pixels to it.

Where Used?

When the camera is fixed/static and you want to detect motion.

Reference Code Skeleton

import cv2

# 1. Initialize Webcam (Index 0 is usually the default cam)
cap = cv2.VideoCapture(0)

# 2. Load Haar Cascade Classifier for plain frontal faces
# Ensure the xml file is in your directory
face_cascade = cv2.CascadeClassifier("haarcascade_frontalface.xml")

attendance_marked = False

while True:
    # 3. Read frame-by-frame
    ret, frame = cap.read()
    if not ret:
        break # Exit if frame not captured

    # 4. Convert to Grayscale (Required for Haar Cascades)
    gray = cv2.cvtColor(frame, cv2.COLOR_BGR2GRAY)

    # 5. Detect Faces
    # scaleFactor=1.3: Image size reduction at each scale
    # minNeighbors=5: Minimum neighbours to retain a window
    faces = face_cascade.detectMultiScale(gray, 1.3, 5)

    # 6. Draw Bounding Boxes
    for (x,y,w,h) in faces:
        # Draw green rectangle around face
        cv2.rectangle(frame, (x,y), (x+w,y+h), (0,255,0), 2)
        
        # 7. Mock Attendance Logic
        if not attendance_marked:
            print(f"Face Detected at x={x}, y={y}. Attendance marked!")
            attendance_marked = True # Avoid repeating every frame

    # 8. Display Result
    cv2.imshow("Face Attendance System", frame)

    # 9. Exit on 'ESC' Key
    if cv2.waitKey(1) == 27:
        break

# 10. Release Resources
cap.release()
cv2.destroyAllWindows()

Important Notes

  • Attendance logic should be separated from detection logic.
  • Grayscale conversion is mandatory for Haar cascades.
  • Use frame skipping for better performance.
  • For production, replace Haar with DNN-based face detectors.

This project detects motion in a fixed camera feed and raises an intrusion alert when significant movement is observed. It is commonly used in CCTV and perimeter security systems.

Reference Code Skeleton

import cv2

# 1. Setup Camera
cap = cv2.VideoCapture(0)

# 2. Initialize Background Subtractor (MOG2)
# history=500: Number of previous frames used for background model
# varThreshold=16: Threshold for pixel to be considered 'foreground'
bg_subtractor = cv2.createBackgroundSubtractorMOG2(history=500, varThreshold=16)

# Kernel for noise removal (morphology)
kernel = cv2.getStructuringElement(cv2.MORPH_RECT, (5,5))

while True:
    ret, frame = cap.read()
    if not ret: 
        break

    # 3. Apply Background Subtraction taking the current frame
    # Returns a binary mask (White = Moving, Black = Static)
    fgmask = bg_subtractor.apply(frame)

    # 4. Remove Noise (Shadows/Speckles)
    # Opening removes small white points from the dark background
    fgmask = cv2.morphologyEx(fgmask, cv2.MORPH_OPEN, kernel)

    # 5. Find Contours of moving objects
    contours, _ = cv2.findContours(fgmask, cv2.RETR_EXTERNAL, cv2.CHAIN_APPROX_SIMPLE)

    for cnt in contours:
        # 6. Filter by Area (Ignore small movements like bugs/leaves)
        if cv2.contourArea(cnt) < 800:
            continue

        # 7. Draw Alert for valid intrusion
        x, y, w, h = cv2.boundingRect(cnt)
        cv2.rectangle(frame, (x, y), (x+w, y+h), (0, 0, 255), 2)
        
        # Add visual label
        cv2.putText(frame, "** INTRUSION DETECTED **", (x, y-10),
                    cv2.FONT_HERSHEY_SIMPLEX, 0.6, (0, 0, 255), 2)

    cv2.imshow("CCTV Intrusion Detection", frame)
    # Also show the mask for debugging
    # cv2.imshow("Mask", fgmask)

    if cv2.waitKey(1) == 27:
        break

cap.release()
cv2.destroyAllWindows()

Practical Notes

  • Camera must be static for reliable results.
  • Contour area threshold filters small noise.
  • Morphology is critical for stability.
  • Lighting changes may cause false positives.

Project: Fire Detection System

This project detects fire in a video stream using colour-based segmentation and contour analysis. It is suitable for early warning systems in CCTV and industrial environments.

System Flow

Video Stream
 → Colour Space Conversion (HSV)
 → Fire Colour Mask
 → Morphology Cleanup
 → Contour Detection
 → Area + Persistence Check
 → Fire Alert
 → Visual Overlay

Key OpenCV APIs Used

Stage Function Purpose
Input cv2.VideoCapture Read camera feed
Colour cv2.cvtColor Convert BGR → HSV
Segmentation cv2.inRange Extract fire-like colours
Cleanup cv2.morphologyEx Remove noise
Detection cv2.findContours Find fire regions

Reference Code Skeleton

import cv2
import numpy as np

cap = cv2.VideoCapture(0)

# 1. Define Fire Colour Range (HSV)
# Adjust these values based on environment lighting
lower_fire = np.array([18, 50, 50])  # Lower limit (Yellow/Orange)
upper_fire = np.array([35, 255, 255]) # Upper limit (Red/Orange)

# Kernel for smoothening the mask
kernel = cv2.getStructuringElement(cv2.MORPH_ELLIPSE, (5,5))
fire_frames = 0 # Counter for persistence check

while True:
    ret, frame = cap.read()
    if not ret: break

    # 2. Convert BGR to HSV for robust colour detection
    hsv = cv2.cvtColor(frame, cv2.COLOR_BGR2HSV)
    
    # 3. Create Binary Mask (White = Fire colour, Black = Rest)
    mask = cv2.inRange(hsv, lower_fire, upper_fire)

    # 4. Refine Mask (Morphology)
    # Open: Removes small noise dots
    # Dilate: Expands the remaining fire regions to fill gaps
    mask = cv2.morphologyEx(mask, cv2.MORPH_OPEN, kernel)
    mask = cv2.morphologyEx(mask, cv2.MORPH_DILATE, kernel)

    # 5. Find Contours of fire regions
    contours, _ = cv2.findContours(mask, cv2.RETR_EXTERNAL, cv2.CHAIN_APPROX_SIMPLE)
    
    fire_detected_this_frame = False

    for cnt in contours:
        # Area filter: ignore small sparks or noise
        if cv2.contourArea(cnt) < 1000:
            continue

        x,y,w,h = cv2.boundingRect(cnt)
        cv2.rectangle(frame, (x,y), (x+w,y+h), (0,0,255), 2)
        fire_detected_this_frame = True

    # 6. Persistence Check (Debouncing)
    # Only alert if fire is detected for >10 consecutive frames
    if fire_detected_this_frame:
        fire_frames += 1
    else:
        fire_frames = 0

    if fire_frames > 10:
        cv2.putText(frame, "!!! FIRE DETECTED !!!", (20,40),
                    cv2.FONT_HERSHEY_SIMPLEX, 1, (0,0,255), 3)

    cv2.imshow("Fire Detection System", frame)

    if cv2.waitKey(1) == 27:
        break

cap.release()
cv2.destroyAllWindows()

Practical Notes

  • HSV is more stable than RGB for fire detection.
  • Area filtering removes sparks and reflections.
  • Persistence (multiple frames) reduces false alarms.
  • Sunlight and yellow lights may cause false positives.
  • For production, combine with motion or DNN-based fire models.

Project: ANPR / Number Plate Detection

This project detects vehicle number plates using classical image processing techniques based on edges, contours, and geometric filtering.

Reference Code Skeleton

import cv2

# 1. Read the input image
img = cv2.imread("car_plate.jpg")

# 2. Convert to Grayscale
# Simplifies the image for edge detection
gray = cv2.cvtColor(img, cv2.COLOR_BGR2GRAY)

# 3. Noise Reduction
# Gaussian Blur removes high-frequency noise
blur = cv2.GaussianBlur(gray, (5,5), 0)

# 4. Edge Detection (Canny)
# Finds structural edges
edges = cv2.Canny(blur, 100, 200)

# 5. Find Contours
contours, _ = cv2.findContours(
    edges, cv2.RETR_EXTERNAL, cv2.CHAIN_APPROX_SIMPLE
)

for cnt in contours:
    # 6. Get Bounding Box
    x, y, w, h = cv2.boundingRect(cnt)
    
    # 7. Geometric Filtering (Heuristics)
    # Number plates have specific aspect ratios (rectangle) and area
    aspect_ratio = w / float(h)
    area = cv2.contourArea(cnt)

    # Filter: Area must be decent size, Aspect Ratio typical of a plate (2:1 to 5:1)
    if area > 2000 and 2 < aspect_ratio < 6:
        # Draw finding
        cv2.rectangle(img, (x, y), (x+w, y+h), (0, 255, 0), 2)
        
        # 8. Crop ROI (Region of Interest)
        # This crop can be sent to Tesseract OCR
        plate_roi = img[y:y+h, x:x+w]
        
        cv2.putText(img, "Plate", (x, y-5), cv2.FONT_HERSHEY_SIMPLEX, 0.5, (0,255,0), 2)

cv2.imshow("ANPR Detection", img)
cv2.waitKey(0)
cv2.destroyAllWindows()

Practical Notes

  • Aspect ratio filtering removes most false positives.
  • Edge detection works best for high-contrast plates.
  • Multiple candidates may appear — ranking is needed.
  • OCR (Tesseract) can be applied on the cropped ROI.
  • For night scenes, add adaptive thresholding.

Project: YOLO-based Object Detection (OpenCV DNN)

This project performs real-time object detection using a pre-trained YOLO model executed via OpenCV’s DNN module. It detects multiple object classes with bounding boxes and confidence scores.

System Flow

Video / Image
 → Resize & Normalise
 → Blob Creation
 → DNN Forward Pass
 → Confidence Filtering
 → Non-Max Suppression
 → Bounding Boxes + Labels

Key OpenCV APIs Used

Stage Function Purpose
Model Load cv2.dnn.readNet Load YOLO weights/config
Preprocess blobFromImage Prepare network input
Inference forward Run detection
Filtering cv2.dnn.NMSBoxes Remove duplicate boxes
Visualise rectangle / putText Draw results

Reference Code Skeleton

import cv2
import numpy as np

# 1. Load YOLO Model (Darknet)
# Requires 'yolov3.weights' and 'yolov3.cfg' files
net = cv2.dnn.readNet("yolov3.weights", "yolov3.cfg")

# Load COCO names (classes)
with open("coco.names") as f:
    classes = [line.strip() for line in f]

# Determine the output layers (where we get detections)
layer_names = net.getLayerNames()
out_layers = [layer_names[i-1] for i in net.getUnconnectedOutLayers()]

# 2. Load Image
img = cv2.imread("street.jpg")
h, w = img.shape[:2]

# 3. Create Blob (Preprocess Image)
# - Scale: 1/255 (normalise pixel values)
# - Size: (416, 416) standard YOLO input
# - SwabRB: Convert BGR to RGB
blob = cv2.dnn.blobFromImage(img, 1/255.0, (416,416), swapRB=True, crop=False)

# 4. Forward Pass (Inference)
net.setInput(blob)
outs = net.forward(out_layers)

# 5. Process Output
boxes, confidences, class_ids = [], [], []

for out in outs:
    for det in out:
        # det contains [x, y, w, h, score, class_probs...]
        scores = det[5:]
        class_id = np.argmax(scores)
        conf = scores[class_id]

        if conf > 0.5: # Confidence threshold
            # YOLO returns relative coordinates (0-1), convert to pixels
            cx, cy, bw, bh = (det[0:4] * [w,h,w,h]).astype(int)
            x = int(cx - bw/2)
            y = int(cy - bh/2)

            boxes.append([x,y,int(bw),int(bh)])
            confidences.append(float(conf))
            class_ids.append(class_id)

# 6. Non-Max Suppression (NMS)
# Eliminates overlapping boxes for the same object
idxs = cv2.dnn.NMSBoxes(boxes, confidences, 0.5, 0.4)

# 7. Draw Final Results
if len(idxs) > 0:
    for i in idxs.flatten():
        x, y, bw, bh = boxes[i]
        label = f"{classes[class_ids[i]]}: {confidences[i]:.2f}"
        
        cv2.rectangle(img, (x,y), (x+bw,y+bh), (0,255,0), 2)
        cv2.putText(img, label, (x, y-5), cv2.FONT_HERSHEY_SIMPLEX, 0.5, (0,255,0), 2)

cv2.imshow("YOLO DNN Detection", img)
cv2.waitKey(0)

Practical Notes

  • YOLOv3/v4 run well on CPU; v5/v8 require ONNX.
  • Lower confidence increases recall but adds noise.
  • NMS is mandatory to avoid duplicate boxes.
  • Resize input early to improve FPS.
  • For production, use ONNX + FP16 where possible.

Project: YOLO Object Detection (ONNX Optimised)

This project performs high-performance object detection using a YOLO model exported to ONNX format and executed via OpenCV’s DNN module. It is the recommended approach for CPU and edge deployments.

System Flow

Image / Video
 → Resize & Normalise
 → Blob Creation
 → ONNX Network Forward
 → Confidence Filtering
 → Non-Max Suppression
 → Bounding Boxes + Labels

Key OpenCV APIs Used

Stage Function Purpose
Model Load cv2.dnn.readNetFromONNX Load ONNX model
Preprocess cv2.dnn.blobFromImage Prepare input tensor
Inference forward Run ONNX graph
Post-process cv2.dnn.NMSBoxes Suppress duplicates

Reference Code Skeleton

import cv2
import numpy as np

# 1. Load ONNX Model
# Best for modern YOLOv8/v9/v10
net = cv2.dnn.readNetFromONNX("yolov8n.onnx")

img = cv2.imread("traffic.jpg")
h, w = img.shape[:2]

# 2. Preprocessing (Blob)
# YOLOv8 expects RGB, size often 640x640, normalized 0-1
blob = cv2.dnn.blobFromImage(
    img,
    scalefactor=1/255.0, 
    size=(640, 640), 
    swapRB=True, 
    crop=False
)

# 3. Inference
net.setInput(blob)
# Forward pass - getting the output tensor
outputs = net.forward()[0]
# Transpose if needed: YOLOv8 ONNX output shape is typically (84, 8400) -> Needs (8400, 84) for iteration
outputs = outputs.transpose() 

boxes, scores, class_ids = [], [], []

# 4. Parse Detections
for det in outputs:
    # det format: [cx, cy, w, h, class_score_1, class_score_2, ...]
    # We find the max score among classes
    class_probs = det[4:]
    class_id = np.argmax(class_probs)
    score = class_probs[class_id]

    if score > 0.5: # Confidence threshold
        cx, cy, bw, bh = det[:4]
        
        # Scaling back to original image size
        x = int((cx - bw/2) * w) # x is normalized, scale by width (correction depends on export)
        # Note: If blob size != img size, rescaling factors are needed.
        # Here assuming simple scaling for demo clarity.
        
        # Actually in standard YOLOv8 export, coords are absolute to 640x640 usually.
        # But let's assume normalized for this broad example or handle via scale factor.
        # Simplified for handbook brevity:
        x = int((cx - bw/2) * (w / 640))
        y = int((cy - bh/2) * (h / 640))
        width = int(bw * (w / 640))
        height = int(bh * (h / 640))

        boxes.append([x, y, width, height])
        scores.append(float(score))
        class_ids.append(class_id)

# 5. Non-Max Suppression
idxs = cv2.dnn.NMSBoxes(boxes, scores, 0.5, 0.45)

# 6. Mark Results
for i in idxs.flatten():
    x, y, bw, bh = boxes[i]
    cv2.rectangle(img, (x, y), (x+bw, y+bh), (0, 255, 0), 2)
    cv2.putText(img, f"Obj {scores[i]:.2f}", (x, y-10), cv2.FONT_HERSHEY_SIMPLEX, 0.5, (0, 255, 0), 2)

cv2.imshow("YOLO ONNX Detection", img)
cv2.waitKey(0)

Practical Notes

  • YOLOv8 ONNX models run faster than Darknet configs.
  • Use smaller models (n, s) for CPU and edge devices.
  • Input size must match export size (e.g. 640×640).
  • ONNX enables TensorRT and OpenVINO acceleration.
  • Post-processing varies slightly per YOLO version.
