1. What is Pandas?

🧠 Mental Model

Think of Pandas as Excel on steroids — a Python library that gives you spreadsheet-like capabilities but with programmable power. Where Excel fails at 1M rows, Pandas handles millions. Where manual clicks slow you down, Pandas scripts automate everything.

💡 What Is It?

Open-source data manipulation library. Two core structures: Series (1D) and DataFrame (2D table). Built on NumPy for speed. The de facto standard for tabular data in Python.

🔧 Why Was It Needed?

Before Pandas, Python lacked proper tabular data handling. You'd use nested lists or dicts — clumsy and slow. Pandas gave Python what R had with data.frames: labelled, intuitive, fast data manipulation.

⚙️ How It Works?

Built on NumPy arrays for vectorised operations. Data stored column-wise (columnar format). Index provides O(1) label lookups. Most operations return new objects (immutable-ish pattern).

🎯 Real-World Use Cases
  • Finance: Portfolio analysis, risk metrics
  • Cybersecurity: Log parsing, threat detection
  • Analytics: ETL pipelines, reporting
  • ML: Feature engineering, data prep
✅ When To Use

Tabular data under ~10GB. Exploratory analysis. Data cleaning. Feature engineering. CSV/Excel/SQL workflows. When you need labelled axes.

⚠️ When NOT To Use

Real-time streaming (use Kafka/Flink). Huge datasets >10GB (use Dask/Polars/Spark). Simple arrays (use NumPy). Graph data (use NetworkX).

Simple Example

import pandas as pd

# The convention: pd is the universal alias
print(pd.__version__)  # Check your version

# Quick taste: create data, filter, aggregate
df = pd.DataFrame({
    'product': ['Widget', 'Gadget', 'Widget', 'Gadget'],
    'sales': [100, 200, 150, 250]
})

# One-liner: total sales per product
print(df.groupby('product')['sales'].sum())
2.1.0 product Gadget 450 Widget 250 Name: sales, dtype: int64

Production Example: Log Analysis

import pandas as pd

# Read server access log (millions of rows)
df = pd.read_csv(
    'access.log',
    sep=r'\s+',
    names=['ip', 'timestamp', 'method', 'path', 'status', 'bytes'],
    parse_dates=['timestamp']
)

# Top 10 IPs by request count (potential attackers?)
suspicious = (
    df.groupby('ip')
    .size()
    .sort_values(ascending=False)
    .head(10)
)

# 4xx/5xx errors per hour
errors = (
    df[df['status'] >= 400]
    .set_index('timestamp')
    .resample('H')
    .size()
)

print("Suspicious IPs:\n", suspicious)
print("\nErrors per hour:\n", errors.head())

❌ Common Mistakes

1. Not using pd alias: Always import pandas as pd. It's the universal convention — deviate and colleagues will be confused.

2. Forgetting Pandas is memory-hungry: A 1GB CSV can take 2-5GB RAM. Always check df.info(memory_usage='deep').

3. Mutating in-place assumption: Most Pandas operations return new objects. df.drop('col') doesn't modify df — you need df = df.drop('col') or df.drop('col', inplace=True).

✅ Quick Revision

  • import pandas as pd — always this convention
  • Two core structures: Series (1D) and DataFrame (2D)
  • Built on NumPy, columnar storage, O(1) label lookups
  • Best for tabular data <10GB
  • Most operations return new objects (not in-place)
💡 Pro Tip: Before any analysis, always run df.info() and df.describe(). These two commands tell you 80% of what you need to know about your data: types, missing values, distributions.

2. Series: The Atomic Unit

🧠 Mental Model

Think of a Series as a single column from a spreadsheet — but with a name tag (index) attached to every cell. It's the building block of DataFrames. When you select one column from a DataFrame, you get a Series. Master this, and DataFrames become intuitive.

One-Dimensional Labeled Array

Series is 1D labeled array. Like a column in spreadsheet. Has index (labels) and values. Can hold any data type. Index allows label-based access. Foundation of DataFrame columns.

💡 What Is It?

1D array with labels (index). Like dict but ordered. Can hold any dtype.

🔧 Why It Exists?

Label-based access. Automatic alignment. Missing data handling. Time series support.

⚙️ How It Works?

Index maps labels to values. Fast lookups. Vectorized operations. NumPy-based.

🎯 Where Used?

Single column data. Time series. Building DataFrames. Quick calculations.

✅ When To Use?

1D data with labels. Time series. Column operations. Simple aggregations.

⚠️ When To Avoid?

Multi-dimensional data (use DataFrame). Complex tabular operations. Large 2D datasets.

Simple Example

import pandas as pd
import numpy as np

# Create Series from list
s = pd.Series([10, 20, 30, 40])
print(s)

# Create with custom index
s = pd.Series([10, 20, 30], index=['a', 'b', 'c'])
print(s)

# Create from dict
data = {'a': 10, 'b': 20, 'c': 30}
s = pd.Series(data)
print(s)

# Access values
print(s['a'])      # Label-based
print(s[0])        # Position-based
print(s[['a', 'c']])  # Multiple

# Vectorized operations
print(s * 2)
print(s > 15)

# Common methods
print(s.mean())
print(s.sum())
print(s.max())
0 10 1 20 2 30 3 40 dtype: int64 a 10 b 20 c 30 dtype: int64 10 a 20 c 60 dtype: int64 20.0 60 30

❌ Common Mistakes

1. Index confusion: s[0] vs s['0'] - position vs label. Use .iloc for position always

2. Assignment without copy: s2 = s creates view. Use s.copy() for independent copy

3. Index alignment: Operations align by index, not position. Can create unexpected NaN

✅ Quick Revision

  • pd.Series(data, index=...) - Create Series
  • Label-based: s['label'], position: s.iloc[0]
  • Vectorized ops: s * 2, s > 10
  • Methods: .mean(), .sum(), .std()
  • Operations auto-align by index
  • Missing values represented as NaN

Create Series with months as index and temperatures as values. Calculate mean temp, find hottest month, filter temps > 25°C.

3. DataFrame (Table)

Two-Dimensional Labeled Data Structure

DataFrame is 2D table. Rows and columns with labels. Like Excel spreadsheet or SQL table. Collection of Series sharing index. Most used pandas structure. Powerful for data analysis.

💡 What Is It?

2D table with row/column labels. Dict of Series. Like SQL table or Excel sheet.

🔧 Why It Exists?

Tabular data manipulation. Column/row operations. SQL-like queries. Statistical analysis.

⚙️ How It Works?

Dict of Series. Shared index. Automatic alignment. Column-oriented storage.

🎯 Where Used?

Data analysis, ETL, reporting, ML features, time series, financial data.

✅ When To Use?

Tabular data. Multiple columns. SQL-like operations. Data cleaning/transformation.

⚠️ When To Avoid?

Huge data (>1GB - use Dask). Real-time streaming. Simple 1D data (use Series).

Simple Example

import pandas as pd

# Create from dict
data = {
    'name': ['Alice', 'Bob', 'Charlie'],
    'age': [25, 30, 35],
    'city': ['NYC', 'LA', 'Chicago']
}
df = pd.DataFrame(data)
print(df)

# Basic info
print(df.shape)      # (3, 3)
print(df.columns)    # Column names
print(df.index)      # Row labels
print(df.dtypes)     # Data types

# Select columns
print(df['name'])           # Single column (Series)
print(df[['name', 'age']])  # Multiple (DataFrame)

# Select rows
print(df.iloc[0])         # First row
print(df.iloc[0:2])       # Slice

# Add column
df['salary'] = [50000, 60000, 70000]

# Summary statistics
print(df.describe())
print(df.info())

  • df[['col1', 'col2']] - Multiple columns

  • df.loc[0, 'col'] - Cell by label
  • .describe(), .info() for overview

4. Data Types (dtypes)

🧠 Mental Model

Every column in a DataFrame has a dtype that determines how memory is allocated and what operations are valid. Think of it like choosing the right container: storing integers as strings wastes memory and slows operations. Getting dtypes right is the first optimisation step.

💡 What Is It?

Data type of a column/Series. Determines memory layout, valid operations, and performance. Inherited from NumPy with Pandas-specific extensions.

🔧 Why Was It Needed?

Wrong dtypes waste memory (strings for numbers), cause errors (arithmetic on strings), and slow operations. Proper typing enables vectorised operations.

⚙️ How It Works?

NumPy dtypes: int64, float64, bool, object. Pandas extensions: datetime64, category, Int64 (nullable), string. Each has different memory footprint.

🎯 Real-World Use Cases
  • Finance: Ensure prices are float, not object
  • Logs: Parse timestamps as datetime64
  • Surveys: Use category for repeated text values
✅ When To Use

Always check dtypes after loading data. Convert early in your pipeline. Use df.dtypes and df.info() religiously.

⚠️ When NOT To Use

Don't force dtypes blindly — understand your data first. Don't use int for columns with missing values (use nullable Int64 or float).

Common dtypes Reference

dtype Description Example Memory (per value)
int64 Integer 1, 42, -100 8 bytes
float64 Decimal 3.14, -0.001 8 bytes
bool True/False True, False 1 byte
object Python objects (usually strings) "hello" Variable (slow)
datetime64[ns] Timestamps 2024-01-15 09:30:00 8 bytes
category Categorical (enum-like) "low", "medium", "high" Depends on cardinality
Int64 Nullable integer 1, 42, NA 8+ bytes
string Pandas string type "text" Variable

Simple Example

import pandas as pd

df = pd.DataFrame({
    'id': ['001', '002', '003'],      # Loaded as object (string)
    'price': ['10.5', '20.0', '15.3'],  # Also object — problematic!
    'count': [100, 200, 150]
})

print(df.dtypes)
# id       object
# price    object   ← Should be float!
# count    int64

# Fix dtypes
df['price'] = df['price'].astype('float64')
print(df.dtypes)
# price    float64  ← Fixed!

# Downcast to save memory
df['count'] = pd.to_numeric(df['count'], downcast='integer')
print(df.dtypes)
# count    int16   ← Smaller, faster!

Production Example: Memory Reduction

import pandas as pd

# Load large CSV
df = pd.read_csv('transactions.csv')
print(f"Memory before: {df.memory_usage(deep=True).sum() / 1e6:.1f} MB")

# Optimise dtypes
dtype_map = {
    'user_id': 'int32',           # Smaller int
    'amount': 'float32',          # Single precision
    'category': 'category',       # Categorical encoding
    'status': 'category',
    'timestamp': 'datetime64[ns]'
}

for col, dtype in dtype_map.items():
    if col in df.columns:
        df[col] = df[col].astype(dtype)

print(f"Memory after: {df.memory_usage(deep=True).sum() / 1e6:.1f} MB")
# Typical reduction: 50-80%!

❌ Common Mistakes

1. Ignoring 'object' dtype: Object columns are slow and memory-hungry. Convert to proper types immediately.

2. Using int for columns with missing values: NaN is float, so int columns with NaN become float64. Use nullable Int64 instead.

3. Not using category for low-cardinality strings: If a column has <50 unique values, category dtype saves massive memory.

✅ Quick Revision

  • df.dtypes — check column types
  • df['col'].astype('float64') — convert type
  • pd.to_numeric(), pd.to_datetime() — smart conversion
  • Use category for repeated strings
  • Use Int64 (capital I) for nullable integers

5. Indexing & Selection

🧠 Mental Model

Think of .loc as using labels/names (like looking up a book by title), and .iloc as using positions/numbers (like finding the 3rd book on a shelf). This distinction is fundamental — mix them up and you'll get unexpected results.

💡 What Is It?

Methods to select rows, columns, or cells from DataFrames/Series. Three main approaches: [], .loc[] (label-based), .iloc[] (position-based).

🔧 Why Was It Needed?

Need to extract subsets of data for analysis. SQL has SELECT/WHERE; Pandas has loc/iloc. Flexible selection enables filtering, slicing, and targeted operations.

⚙️ How It Works?

.loc[] uses index labels, .iloc[] uses integer positions. Both accept: single value, list, slice, or boolean array.

🎯 Real-World Use Cases
  • Analytics: Select specific customers by ID
  • Finance: Get price for specific date
  • Logs: Extract first N rows for preview
✅ When To Use

Use .loc when you know labels. Use .iloc for position-based access. Use [] for simple column selection.

⚠️ When NOT To Use

Avoid df['col'][0] (chained indexing). Don't use .loc when you mean .iloc and vice versa.

Quick Reference

Method Syntax What It Returns
df['col'] df['name'] Series
df[['col']] df[['name', 'age']] DataFrame
.loc[row] df.loc['Alice'] Row by label
.loc[row, col] df.loc['Alice', 'age'] Cell by labels
.iloc[pos] df.iloc[0] Row by position
.iloc[pos, pos] df.iloc[0, 1] Cell by position
.at[row, col] df.at['Alice', 'age'] Fast scalar (label)
.iat[pos, pos] df.iat[0, 1] Fast scalar (position)

Simple Example

import pandas as pd

df = pd.DataFrame({
    'name': ['Alice', 'Bob', 'Charlie'],
    'age': [25, 30, 35],
    'city': ['London', 'Paris', 'Berlin']
}, index=['a', 'b', 'c'])

# Column selection
print(df['name'])              # Series
print(df[['name', 'age']])     # DataFrame

# .loc — label-based
print(df.loc['a'])             # Row 'a'
print(df.loc['a', 'name'])     # 'Alice'
print(df.loc['a':'b'])          # Rows a AND b (inclusive!)

# .iloc — position-based
print(df.iloc[0])              # First row
print(df.iloc[0, 1])           # First row, second column (25)
print(df.iloc[0:2])            # First two rows (exclusive end!)

# Fast scalars
print(df.at['a', 'age'])       # 25 (fastest for single value)
print(df.iat[0, 1])            # 25 (position-based)
⚠️ Critical Difference: .loc slices are inclusive on both ends: df.loc['a':'c'] includes 'c'. But .iloc slices are exclusive on the end: df.iloc[0:2] gives positions 0 and 1 only.

❌ Common Mistakes

1. Chained indexing: df['col'][0] = val may not work. Use df.loc[0, 'col'] = val instead.

2. Mixing loc and iloc: df.loc[0] looks for label 0, not position 0. If your index is 'a', 'b', 'c', this fails.

3. Forgetting inclusive/exclusive slicing: loc includes end, iloc excludes it.

✅ Quick Revision

  • df['col'] — quick column access
  • .loc[] — label-based, inclusive slices
  • .iloc[] — position-based, exclusive slices
  • .at[]/.iat[] — fast single-value access
  • Avoid chained indexing

6. Boolean Indexing (Filtering)

🧠 Mental Model

Boolean indexing is like a SQL WHERE clause. You create a True/False mask for each row, then pass it to the DataFrame to keep only the True rows. Master this and you'll spend 80% less time trying to filter data.

💡 What Is It?

Using boolean conditions to filter rows. Comparison operations create boolean Series, which then select matching rows from the DataFrame.

🔧 Why Was It Needed?

Need to select subsets based on conditions. "Give me all sales > £1000" or "Find failed login attempts". Core data analysis operation.

⚙️ How It Works?

Comparison returns boolean Series. Pass to df[] or .loc[]. Combine with & (and), | (or), ~ (not).

🎯 Real-World Use Cases
  • Finance: Transactions above threshold
  • Cyber: Failed login attempts
  • Analytics: Users in specific region
✅ When To Use

Any time you need to filter rows based on column values. Combine with .loc to filter and select specific columns.

⚠️ When NOT To Use

Complex nested conditions — use .query() for readability. Very large DataFrames — consider chunking first.

Simple Example

import pandas as pd

df = pd.DataFrame({
    'name': ['Alice', 'Bob', 'Charlie', 'Diana'],
    'age': [25, 30, 35, 28],
    'city': ['London', 'Paris', 'London', 'Berlin']
})

# Single condition
print(df[df['age'] > 28])               # Age over 28

# Multiple conditions — use & (and), | (or)
# IMPORTANT: wrap each condition in parentheses!
print(df[(df['age'] > 25) & (df['city'] == 'London')])

# Negate with ~
print(df[~(df['city'] == 'London')])       # NOT London

# isin() for multiple values
print(df[df['city'].isin(['London', 'Paris'])])

# String methods with .str
print(df[df['name'].str.startswith('A')])   # Names starting with A

# Filter AND select columns
print(df.loc[df['age'] > 28, ['name', 'age']])

Production Example: Security Logs

import pandas as pd

# Load security events
logs = pd.read_csv('security_events.csv', parse_dates=['timestamp'])

# Assume known_ips is a pre-defined list or Series of trusted IPs
known_ips = ['192.168.1.1', '10.0.0.1'] 

# Failed login attempts from unknown IPs
suspicious = logs[
    (logs['event_type'] == 'LOGIN_FAILED') &
    (~logs['src_ip'].isin(known_ips)) &
    (logs['timestamp'] > '2024-01-01')
]

# Brute force detection: IPs with >10 failures
brute_force = (
    suspicious.groupby('src_ip')
    .size()
    .reset_index(name='attempts')
    .query('attempts > 10')
)

print(f"Potential brute force attacks: {len(brute_force)}")

❌ Common Mistakes

1. Using 'and'/'or' instead of '&'/'|': Python's and/or don't work with Series. Use & and |.

2. Forgetting parentheses: df[df['a'] > 1 & df['b'] < 2] fails. Must be df[(df['a'] > 1) & (df['b'] < 2)].

3. Comparing to None: Use df['col'].isna() not df['col'] == None.

✅ Quick Revision

  • df[df['col'] > x] — basic filter
  • & (and), | (or), ~ (not) — combine conditions
  • Always wrap conditions in parentheses
  • .isin() — check membership in list
  • .isna() / .notna() — check for missing

7. Reading CSV

🧠 Mental Model

pd.read_csv() is your gateway to data. It's the most common way data enters Pandas. Think of it as a highly configurable file reader — you can handle almost any CSV quirk with the right parameters. Know 5-6 key parameters and you'll handle 95% of cases.

💡 What Is It?

Function to load CSV/text files into DataFrames. Handles delimiters, encoding, dates, missing values, data types — all configurable.

🔧 Why Was It Needed?

CSV is the lingua franca of data. Every system exports it. Pandas needed robust, flexible CSV parsing with sensible defaults and extensive customisation.

⚙️ How It Works?

Reads file line-by-line, infers types, builds DataFrame. Can read from files, URLs, S3, or any file-like object. Supports chunked reading for large files.

🎯 Real-World Use Cases
  • ETL: Load data from exports
  • Analytics: Import spreadsheet data
  • Logs: Parse structured log files
✅ When To Use

Loading CSV, TSV, or any delimited text. Specify dtypes upfront for memory efficiency. Use nrows for previewing large files.

⚠️ When NOT To Use

For huge files (>1GB), use chunking or Parquet. For binary/database data, use pd.read_sql() or pd.read_parquet().

Key Parameters Reference

Parameter Purpose Example
sep Delimiter character sep='\t' for TSV
header Row for column names header=None if no header
names Column names list names=['a','b','c']
usecols Only load specific columns usecols=['id','name']
dtype Force column types dtype={'id': 'int32'}
parse_dates Parse date columns parse_dates=['date']
na_values Treat as missing na_values=['N/A', '-']
nrows Read only N rows nrows=1000
chunksize Read in chunks chunksize=10000
encoding File encoding encoding='utf-8'

Simple Example

import pandas as pd

# Basic read
df = pd.read_csv('data.csv')

# Tab-separated, custom columns
df = pd.read_csv(
    'data.tsv',
    sep='\t',
    names=['id', 'name', 'value'],
    header=0  # Use first row, but replace with names
)

# Read only specific columns with types
df = pd.read_csv(
    'sales.csv',
    usecols=['date', 'product', 'amount'],
    dtype={'product': 'category', 'amount': 'float32'},
    parse_dates=['date']
)

# Preview large file
preview = pd.read_csv('huge.csv', nrows=100)

Production Example: Large File Processing

import pandas as pd

# Process 10GB file in chunks
chunk_results = []

for chunk in pd.read_csv('massive_log.csv', chunksize=100000):
    # Filter and aggregate each chunk
    filtered = chunk[chunk['status'] == 'ERROR']
    summary = filtered.groupby('source').size()
    chunk_results.append(summary)

# Combine all chunk summaries
total = pd.concat(chunk_results).groupby(level=0).sum()
print(total.sort_values(ascending=False).head(10))

❌ Common Mistakes

1. Not specifying dtypes: Pandas guesses types, often wrong (ints become float64). Specify dtypes for memory and accuracy.

2. Loading entire huge file: Use chunksize or nrows for files that don't fit in memory.

3. Encoding errors: Default is UTF-8. If you see weird characters, try encoding='latin-1' or encoding='cp1252'.

✅ Quick Revision

  • pd.read_csv('file.csv') — basic read
  • usecols — load only needed columns
  • dtype — specify types for efficiency
  • parse_dates — convert date columns
  • chunksize — process large files in chunks

8. Reading Excel

🧠 Mental Model

Think of pd.read_excel() as a universal Excel translator — it can read .xls, .xlsx files, handle multiple sheets, skip rows, and parse specific ranges. Unlike CSV, Excel files are binary formats that require an engine (openpyxl or xlrd).

💡 What Is It?

Function to read Excel files into DataFrame. Supports .xlsx (openpyxl) and .xls (xlrd). Can read specific sheets, ranges, and handle formatting.

🔧 Why It Exists?

Excel is the universal business format. Need to import spreadsheets for analysis. Must handle multiple sheets, formulas, dates, and formatting.

⚙️ How It Works?

Uses openpyxl/xlrd engines to parse binary format. Converts Excel data types to pandas dtypes. Can return dict of DataFrames for multiple sheets.

🎯 Where Used?

Business reports, financial data, inventory exports, HR data, any Excel-based workflow.

✅ When To Use?

Reading .xlsx/.xls files. Multiple sheets needed. Need to preserve dates/numbers from Excel.

⚠️ When To Avoid?

Very large files (>100MB — slow). CSV works fine. Real-time processing. No Excel dependency wanted.

Simple Example

import pandas as pd

# Basic Excel read
df = pd.read_excel('data.xlsx')

# Read specific sheet by name
df = pd.read_excel('data.xlsx', sheet_name='Sales')

# Read specific sheet by index (0-based)
df = pd.read_excel('data.xlsx', sheet_name=0)

# Read ALL sheets — returns dict of DataFrames
all_sheets = pd.read_excel('data.xlsx', sheet_name=None)
print(all_sheets.keys())  # dict_keys(['Sheet1', 'Sales', 'Inventory'])
df_sales = all_sheets['Sales']

# Skip rows and use specific columns
df = pd.read_excel('data.xlsx', 
                   skiprows=2,           # Skip first 2 rows
                   usecols='A:D',        # Excel-style range
                   nrows=100)            # Read only 100 rows

# Parse dates and set index
df = pd.read_excel('data.xlsx',
                   parse_dates=['Date'],
                   index_col='Date')

# Specify engine explicitly
df = pd.read_excel('old_file.xls', engine='xlrd')    # For .xls
df = pd.read_excel('new_file.xlsx', engine='openpyxl')  # For .xlsx

❌ Common Mistakes

1. Missing engine: For .xlsx need openpyxl installed. pip install openpyxl

2. Wrong sheet name: Sheet names are case-sensitive. Use sheet_name=None first to see all names.

3. Date parsing: Excel dates can be weird numbers. Use parse_dates=['col'] to convert properly.

✅ Quick Revision

  • pd.read_excel('file.xlsx') — basic read
  • sheet_name='Sheet1' or sheet_name=0
  • sheet_name=None — read ALL sheets as dict

11. Parquet Files

🧠 Mental Model

Parquet is the performance king of data formats. Unlike CSV (row-based), Parquet stores data column-by-column with compression. This means: 10x smaller files, 100x faster reads for analytics queries that only need specific columns. The modern standard for data engineering.

💡 What Is It?

Columnar storage format from Apache. Binary, compressed, schema-aware. Ideal for analytical workloads and big data tools.

🔧 Why It Exists?

CSV is slow and bloated. Need efficient storage for large datasets. Analytics queries only read specific columns — columnar format excels here.

⚙️ How It Works?

Data stored by column, not row. Each column compressed separately. Schema embedded in file. Supports nested data and partitioning.

🎯 Where Used?

Data lakes, Spark/Databricks, AWS Athena, data warehouses, ML pipelines, any big data workflow.

✅ When To Use?

Large datasets (>100MB). Analytical queries. Data pipelines. Cloud storage. When you need speed.

⚠️ When To Avoid?

Small data (use CSV). Need human-readable format. Row-based access patterns. Quick sharing.

Simple Example

import pandas as pd

# Read parquet file

# Read only specific columns (FAST — only reads needed data!)
df = pd.read_parquet('data.parquet', columns=['name', 'age'])

# Read from cloud storage
df = pd.read_parquet('s3://bucket/data.parquet')  # AWS S3
df = pd.read_parquet('gs://bucket/data.parquet')  # Google Cloud

# Write parquet file
df.to_parquet('output.parquet')

# Choose compression (snappy is default, gzip for smaller files)
df.to_parquet('output.parquet', compression='gzip')
df.to_parquet('output.parquet', compression='snappy')  # Fast compression

# Choose engine: 'pyarrow' (default), 'fastparquet'
df.to_parquet('output.parquet', engine='pyarrow')

df.to_parquet(
    'output_dir',
    partition_cols=['year', 'month']  # Creates folder structure
)

# Compare file sizes
df.to_csv('data.csv')       # ~100 MB
df.to_parquet('data.parquet') # ~10 MB (10x smaller!)

❌ Common Mistakes

1. Missing engine: Install pyarrow: pip install pyarrow

2. Column types: Parquet is strict about types. Mixed types in columns will fail.

3. Object columns: String columns as 'object' dtype can be slow. Convert to 'string' dtype.

✅ Quick Revision

  • pd.read_parquet('file.parquet') — read
  • columns=['a', 'b'] — read only needed columns
  • df.to_parquet('file.parquet') — write
  • compression='gzip' — smaller size
  • compression='snappy' — faster read/write

12. Missing Data

🧠 Mental Model

Missing data is everywhere in real datasets. Pandas uses NaN (Not a Number) and None to represent missing values. Your job: detect them, decide whether to fill them (impute) or drop them, then clean your data before analysis.

💡 What Is It?

Missing values represented as NaN (float) or NA (nullable). Methods to detect, drop, or fill missing data in Series/DataFrames.

🔧 Why Was It Needed?

Real data is messy — sensors fail, users skip fields, exports have gaps. Need consistent way to represent and handle missing values.

⚙️ How It Works?

NaN is a float, propagates through calculations. isna() detects, dropna() removes, fillna() replaces.

🎯 Real-World Use Cases
  • Sensors: Fill gaps with interpolation
  • Surveys: Drop incomplete responses
  • Finance: Forward-fill prices for holidays
✅ When To Use

Always check for missing data early. Choose strategy based on context: drop if data is abundant, fill if patterns exist, leave if algorithms handle NaN.

⚠️ When NOT To Use

Don't blindly drop — you lose information. Don't fill with mean for time series. Don't assume NaN means zero.

Simple Example

import pandas as pd
import numpy as np

df = pd.DataFrame({
    'name': ['Alice', 'Bob', None, 'Diana'],
    'age': [25, np.nan, 35, 28],
    'score': [85, 90, np.nan, np.nan]
})

# Detect missing values
print(df.isna())         # Boolean mask
print(df.isna().sum())  # Count per column

# Drop rows with ANY missing value
print(df.dropna())

# Drop rows missing specific column
print(df.dropna(subset=['name']))

# Fill with specific value
print(df['score'].fillna(0))

# Fill with column mean
print(df['age'].fillna(df['age'].mean()))

# Forward/backward fill (time series)
print(df['score'].fillna(method='ffill'))  # Forward fill
print(df['score'].fillna(method='bfill'))  # Backward fill

Production Example: Sensor Data Cleaning

import pandas as pd

# Load sensor readings with gaps
sensors = pd.read_csv('iot_readings.csv', parse_dates=['timestamp'])
sensors = sensors.set_index('timestamp')

# Report missing data
missing = sensors.isna().sum()
print(f"Missing values:\n{missing[missing > 0]}")

# Strategy: interpolate numeric columns (linear)
numeric_cols = sensors.select_dtypes(include='number').columns
sensors[numeric_cols] = sensors[numeric_cols].interpolate(method='time')

# Forward-fill categorical columns
cat_cols = sensors.select_dtypes(include='object').columns
sensors[cat_cols] = sensors[cat_cols].fillna(method='ffill')

# Drop remaining rows with any NaN
clean = sensors.dropna()
print(f"Retained {len(clean)/len(sensors)*100:.1f}% of data")

❌ Common Mistakes

1. Comparing with == : df['col'] == np.nan is always False. Use df['col'].isna().

2. Filling time series with mean: Destroys temporal patterns. Use ffill(), bfill(), or interpolate().

3. Not checking after cleaning: Always verify with df.isna().sum() after cleaning operations.

✅ Quick Revision

  • df.isna() — detect missing values
  • df.dropna() — remove rows with NaN
  • df.fillna(value) — replace NaN with value
  • fillna(method='ffill') — forward fill
  • df.interpolate() — estimate missing values

🧠 Mental Model

Duplicate rows are data quality killers. They inflate counts, skew averages, and cause join explosions. Pandas provides duplicated() to find them and drop_duplicates() to remove them. Always check for duplicates before analysis.

💡 What Is It?

Methods to identify and remove duplicate rows based on all columns or specific subsets. Returns boolean mask or deduplicated DataFrame.

Data often has duplicates from imports, merges, or data entry errors. Need to identify and handle them for accurate analysis.

Simple Example

16. Outlier Detection

🧠 Mental Model

Outliers are data points that don't belong. They can be errors (typos, sensor malfunctions) or genuine extremes (billionaire in salary data). Two main approaches: IQR method (robust) and Z-score (assumes normal distribution). Detect first, then decide: remove, cap, or investigate.

💡 What Is It?

Statistical methods to identify extreme values. IQR uses quartiles. Z-score uses standard deviations. Quantiles for percentile-based thresholds.

🔧 Why It Exists?

Outliers skew means, break ML models, and distort visualizations. Need systematic way to identify and handle them.

⚙️ How It Works?

IQR: values outside Q1-1.5*IQR to Q3+1.5*IQR. Z-score: values beyond ±3σ. Quantile: beyond custom percentiles.

🎯 Where Used?

Data quality checks, ML preprocessing, fraud detection, anomaly detection, statistical analysis.

✅ When To Use?

Before training models. When means seem off. Cleaning numeric columns. Anomaly detection.

⚠️ When To Avoid?

Non-normal distributions (use IQR). When outliers ARE the signal (fraud). Small datasets.

Simple Example

import pandas as pd
import numpy as np

df = pd.DataFrame({'value': [10, 12, 14, 15, 100, 13, 11, -50]})

# ═══ IQR METHOD (Recommended - robust to non-normal data) ═══
Q1 = df['value'].quantile(0.25)
Q3 = df['value'].quantile(0.75)
IQR = Q3 - Q1

lower = Q1 - 1.5 * IQR
upper = Q3 + 1.5 * IQR

outliers = (df['value'] < lower) | (df['value'] > upper)
print(df[outliers])  # Shows outliers

# Remove outliers
df_clean = df[~outliers]

# ═══ Z-SCORE METHOD (assumes normal distribution) ═══
z_scores = (df['value'] - df['value'].mean()) / df['value'].std()
outliers = z_scores.abs() > 3  # Beyond 3 std devs

# ═══ PERCENTILE METHOD (cap at thresholds) ═══
lower_cap = df['value'].quantile(0.01)
upper_cap = df['value'].quantile(0.99)
df['capped'] = df['value'].clip(lower_cap, upper_cap)

❌ Common Mistakes

1. Always removing: Sometimes outliers are the most important data. Investigate first!

2. Z-score on skewed data: Z-score assumes normal distribution. Use IQR for skewed data.

3. Per-column vs per-row: Multivariate outliers need different techniques (Mahalanobis distance).

✅ Quick Revision

  • IQR: Q1-1.5*IQR to Q3+1.5*IQR (robust)
  • Z-score: Beyond ±3σ (normal data)
  • df['col'].clip(lower, upper) — cap values
  • Investigate before removing!

💪 Practice

17. Apply Functions

🧠 Mental Model

apply() is your escape hatch from vectorization. When pandas' built-in methods can't do what you need, apply() lets you run any custom function row-by-row or column-by-column. It's flexible but slower than vectorized operations — use as last resort.

💡 What Is It?

Method to apply custom functions along Series/DataFrame axes. Works row-wise or column-wise. Accepts any callable.

🔧 Why It Exists?

Built-in methods can't cover every use case. Need way to apply arbitrary logic to data. Bridge between Python functions and pandas.

⚙️ How It Works?

Iterates over axis and calls function on each element/row/column. axis=0 applies to columns, axis=1 to rows.

🎯 Where Used?

Custom transformations, feature engineering, complex row logic, any non-vectorizable operation.

✅ When To Use?

Custom logic not in pandas. Complex row calculations. When vectorization isn't possible.

⚠️ When To Avoid?

When vectorized method exists (use that). Large data (apply is slow). Simple operations.

Simple Example

import pandas as pd

df = pd.DataFrame({'a': [1, 2, 3], 'b': [10, 20, 30]})

# Apply to Series (element-wise)
df['a_squared'] = df['a'].apply(lambda x: x ** 2)

# Apply to DataFrame columns (axis=0, default)
column_sums = df.apply(sum)  # Sum each column

# Apply to DataFrame rows (axis=1)
df['row_sum'] = df.apply(lambda row: row['a'] + row['b'], axis=1)

def categorize(value):
    if value < 15:
        return 'low'
    elif value < 25:
        return 'medium'
    else:
        return 'high'

df['b_category'] = df['b'].apply(categorize)

# With result_type for row operations
result = df.apply(lambda row: (row['a'], row['b']*2), axis=1, result_type='expand')

❌ Common Mistakes

1. Using when vectorized works: df['a'].apply(lambda x: x*2) is slower than df['a'] * 2

2. axis confusion: axis=0 applies to COLUMNS, axis=1 applies to ROWS (opposite of what you'd think)

3. Returning wrong type: Ensure function returns consistent types for all inputs.

✅ Quick Revision

  • series.apply(func) — element-wise
  • df.apply(func, axis=0) — column-wise

18. Map & Transform

🧠 Mental Model

map() and transform() are specialized siblings of apply. map() is for Series only — element-wise mapping from dict, Series, or function. transform() preserves shape — returns same-sized output, crucial for GroupBy operations.

💡 What Is It?

map(): Series method for element-wise substitution. transform(): Returns same-shaped result, works with GroupBy.

🔧 Why It Exists?

map() for simple lookups without apply overhead. transform() to broadcast aggregations back to original shape.

⚙️ How It Works?

map() looks up values in dict/Series or applies function. transform() applies function and broadcasts result to match input shape.

🎯 Where Used?

map(): encoding categories, lookups. transform(): group-level calculations, normalization within groups.

✅ When To Use?

map() for value lookups. transform() when you need same-shape output from GroupBy.

⚠️ When To Avoid?

map() on DataFrame (use applymap). transform() when you want aggregated output (use agg).

Simple Example

import pandas as pd

# ═══ MAP (Series only) ═══
df = pd.DataFrame({'grade': ['A', 'B', 'C', 'A']})

# Map with dict
grade_points = {'A': 4, 'B': 3, 'C': 2}
df['points'] = df['grade'].map(grade_points)

# Map with function
df['lower'] = df['grade'].map(str.lower)

# Map with Series
lookup = pd.Series([4, 3, 2], index=['A', 'B', 'C'])
df['points2'] = df['grade'].map(lookup)

# ═══ TRANSFORM (preserves shape) ═══
df = pd.DataFrame({
    'group': ['A', 'A', 'B', 'B'],
    'value': [10, 20, 30, 40]
})

# Transform with function
df['normalized'] = df['value'].transform(lambda x: (x - x.mean()) / x.std())

# Transform with GroupBy — KEY USE CASE!
df['group_mean'] = df.groupby('group')['value'].transform('mean')
# Each row gets its group's mean (same shape as input!)

df['pct_of_group'] = df['value'] / df.groupby('group')['value'].transform('sum')

❌ Common Mistakes

1. map on DataFrame: map() is Series only. For DataFrame element-wise, use df.applymap() (deprecated) or df.map() (pandas 2.1+)

2. Unmapped values: map() returns NaN for values not in dict. Use fillna() or ensure complete mapping.

3. transform vs agg: transform() must return same shape. agg() returns reduced shape.

✅ Quick Revision

  • series.map(dict) — value lookup
  • series.map(func) — element-wise function
  • df.transform(func) — same-shape output
  • groupby().transform() — broadcast group stats

💪 Practice

19. Merge & Join

🧠 Mental Model

Merge is SQL JOIN for DataFrames. You combine two tables based on matching keys. Think of it like a Venn diagram: inner join keeps overlap, left keeps all from left, etc. This is how you connect related datasets — customers to orders, logs to users.

💡 What Is It?

Combine DataFrames by matching column values. Like SQL JOIN. Supports inner, left, right, outer, and cross joins.

🔧 Why Was It Needed?

Real data is normalised across tables. Need to combine customers with orders, products with categories, users with logs, etc.

⚙️ How It Works?

pd.merge(left, right, on='key') matches rows where key values are equal. how controls join type. left_on/right_on for different column names.

🎯 Real-World Use Cases
  • E-commerce: Orders + Customers + Products
  • Security: Logs + IP reputation data
  • Finance: Trades + Security master
✅ When To Use

Combining datasets with common key. Enriching data with lookups. Any SQL-like join operation.

⚠️ When NOT To Use

Stacking rows vertically — use concat(). Joining on index — use .join().

Join Types Reference

how= Description SQL Equivalent
'inner' Only matching keys from both INNER JOIN
'left' All from left, matches from right LEFT JOIN
'right' All from right, matches from left RIGHT JOIN
'outer' All keys from both, NaN for missing FULL OUTER JOIN
'cross' Every row paired (cartesian) CROSS JOIN

Simple Example

import pandas as pd

# Two related tables
customers = pd.DataFrame({
    'customer_id': [1, 2, 3],
    'name': ['Alice', 'Bob', 'Charlie']
})

orders = pd.DataFrame({
    'order_id': [101, 102, 103, 104],
    'customer_id': [1, 2, 2, 4],  # Note: customer 4 doesn't exist!
    'amount': [100, 200, 150, 50]
})

# Inner join — only matching
print(pd.merge(customers, orders, on='customer_id'))

# Left join — keep all customers
print(pd.merge(customers, orders, on='customer_id', how='left'))

# Different column names
df1['id'] = df1['customer_id']  # Example
print(pd.merge(df1, df2, left_on='id', right_on='customer_id'))

# Merge on multiple columns
print(pd.merge(df1, df2, on=['year', 'month']))

Production Example: Customer Analytics

import pandas as pd

# Load multiple related tables
customers = pd.read_csv('customers.csv')
orders = pd.read_csv('orders.csv', parse_dates=['order_date'])
products = pd.read_csv('products.csv')

# Chain merges: orders + customers + products

    orders
    .merge(customers, on='customer_id', how='left')
    .merge(products, on='product_id', how='left')
)

# Analysis: revenue by customer segment and product category
analysis = (
    full_data
    .groupby(['customer_segment', 'product_category'])
    .agg(

❌ Common Mistakes

1. Duplicate keys explosion: Many-to-many merge creates cartesian product. Check for duplicates with df.duplicated(subset='key') before merging.

2. Forgetting suffixes: Same column names get _x, _y appended. Use suffixes=('_left', '_right') for clarity.

3. Using wrong how: Inner (default) silently drops non-matching rows. Use left join to preserve all rows from main table.

✅ Quick Revision

  • how='left/right/outer/inner' — join type
  • left_on='a', right_on='b' — different column names
  • suffixes — handle duplicate column names
  • Check for duplicates before merging!

20. Concat & Stack

🧠 Mental Model

concat() is gluing DataFrames together — vertically (stacking rows) or horizontally (adding columns). Unlike merge, concat doesn't use keys to match — it just aligns by index. Think "append" but more powerful.

💡 What Is It?

Function to concatenate DataFrames along rows (axis=0) or columns (axis=1). Aligns by index. Can handle mismatched columns.

🔧 Why It Exists?

Need to combine multiple DataFrames that share same structure. Merge is for key-based joins, concat is for simple stacking.

⚙️ How It Works?

Takes list of DataFrames, concatenates along specified axis. Handles different columns via join='inner'/'outer'.

🎯 Where Used?

Combining monthly files, appending new data, stacking train/test sets, batch processing results.

✅ When To Use?

Same-structure DataFrames. No key-based matching needed. Appending rows or adding columns by index.

⚠️ When To Avoid?

When you need key-based matching (use merge). When index alignment isn't desired.

Simple Example

import pandas as pd

df1 = pd.DataFrame({'A': [1, 2], 'B': [3, 4]})
df2 = pd.DataFrame({'A': [5, 6], 'B': [7, 8]})

# Stack vertically (add rows) — axis=0 (default)
combined = pd.concat([df1, df2])
print(combined)
#    A  B
# 0  1  3
# 1  2  4
# 0  5  7  ← Note: index is duplicated!
# 1  6  8

# Reset index to avoid duplicates
combined = pd.concat([df1, df2], ignore_index=True)

# Stack horizontally (add columns) — axis=1
side_by_side = pd.concat([df1, df2], axis=1)

# Handle mismatched columns
df3 = pd.DataFrame({'A': [9], 'C': [10]})  # No column B!
outer = pd.concat([df1, df3], join='outer')  # NaN for missing
inner = pd.concat([df1, df3], join='inner')  # Only common columns

# Add keys to identify source
combined = pd.concat([df1, df2], keys=['file1', 'file2'])

# Concat list of DataFrames (common pattern)
dfs = [pd.read_csv(f) for f in csv_files]
all_data = pd.concat(dfs, ignore_index=True)

❌ Common Mistakes

1. Duplicate index: concat preserves original indices. Use ignore_index=True for clean index.

2. Column mismatch: Different columns create NaN. Use join='inner' if you only want common columns.

3. axis confusion: axis=0 stacks rows (vertical), axis=1 adds columns (horizontal).

✅ Quick Revision

  • pd.concat([df1, df2]) — stack rows
  • axis=1 — add columns side-by-side
  • ignore_index=True — reset index
  • join='inner'/'outer' — handle mismatched columns
  • keys=['a', 'b'] — identify source

21. Pivot Tables

🧠 Mental Model

Pivot tables are Excel-style summaries on steroids. Reshape your data by turning unique values into columns, grouping by rows, and aggregating values. Perfect for creating cross-tabulations and summary reports.

💡 What Is It?

Method to reshape data by creating spreadsheet-style pivot tables. Rows from one column, columns from another, values aggregated.

🔧 Why It Exists?

Raw data is often long-format. Need to summarize and cross-tabulate for analysis and reporting.

⚙️ How It Works?

Groups data by index and columns, applies aggregation function. Creates new DataFrame with unique combinations.

🎯 Where Used?

Business reports, sales summaries, cross-tabulations, multi-dimensional analysis.

✅ When To Use?

Creating summary tables. Cross-tabulating categories. When you need row/column structure.

⚠️ When To Avoid?

Very high cardinality (too many unique values). When you just need groupby aggregations.

Simple Example

import pandas as pd

df = pd.DataFrame({
    'date': ['2024-01', '2024-01', '2024-02', '2024-02'],
    'product': ['A', 'B', 'A', 'B'],
    'sales': [100, 150, 200, 175]
})

# Basic pivot table
pivot = df.pivot_table(
    values='sales',
    index='date',
    columns='product',
    aggfunc='sum'
)
print(pivot)
# product      A    B
# date
# 2024-01    100  150
# 2024-02    200  175

# Multiple aggregation functions
pivot = df.pivot_table(
    values='sales',
    index='date',
    columns='product',
    aggfunc=['sum', 'mean', 'count']
)

# Add margins (totals)
pivot = df.pivot_table(
    values='sales',
    index='date',
    columns='product',
    aggfunc='sum',
    margins=True  # Adds 'All' row and column
)

# Fill missing values
pivot = df.pivot_table(values='sales', index='date', columns='product', fill_value=0)

# Simple crosstab (counts)
counts = pd.crosstab(df['date'], df['product'])

❌ Common Mistakes

1. No aggfunc: When duplicates exist for index/column combo, you must specify aggfunc.

2. pivot() vs pivot_table(): pivot() doesn't aggregate — errors on duplicates. Use pivot_table() for aggregation.

3. Missing combinations: Create NaN by default. Use fill_value=0 to fill.

✅ Quick Revision

  • df.pivot_table(values, index, columns, aggfunc)
  • aggfunc='sum'/'mean'/['sum', 'count']
  • margins=True — add totals
  • fill_value=0 — replace NaN
  • pd.crosstab() — quick frequency counts

22. GroupBy Basics

🧠 Mental Model

GroupBy is split-apply-combine: split data into groups based on column values, apply a function to each group, combine results. It's like SQL's GROUP BY. Master this and you can answer any "by category" question: sales by region, errors by server, avg score by class.

💡 What Is It?

Method to split DataFrame by column values, apply aggregations, and combine results. Core pattern for summarising data by categories.

🔧 Why Was It Needed?

Most analysis is "X by Y": sales by month, users by country, errors by type. GroupBy provides flexible, efficient way to compute grouped statistics.

⚙️ How It Works?

Returns GroupBy object (lazy). Apply aggregations like .sum(), .mean(), .agg(). Can group by multiple columns.

🎯 Real-World Use Cases
  • Sales: Revenue by product category
  • Logs: Error count by service
  • HR: Avg salary by department
✅ When To Use

Any categorical aggregation. Summary statistics by group. Comparing subsets of data. Preparing pivot table data.

⚠️ When To Avoid?

For row-wise transformations, use apply() with axis=1. For simple filtering, use boolean indexing instead.

Simple Example

import pandas as pd

df = pd.DataFrame({
    'region': ['North', 'South', 'North', 'South', 'North'],
    'product': ['A', 'A', 'B', 'B', 'A'],
    'sales': [100, 150, 200, 120, 180]
})

# Basic groupby
print(df.groupby('region')['sales'].sum())

# Multiple aggregations
print(df.groupby('region')['sales'].agg(['sum', 'mean', 'count']))

# Group by multiple columns
print(df.groupby(['region', 'product'])['sales'].sum())

# Named aggregations (modern syntax)
result = df.groupby('region').agg(
    total_sales=('sales', 'sum'),
    avg_sales=('sales', 'mean'),
    num_transactions=('sales', 'count')
)
print(result)

# Reset index to get DataFrame back
print(df.groupby('region')['sales'].sum().reset_index())

Production Example: Log Analysis

import pandas as pd

# Load application logs
logs = pd.read_csv('app_logs.csv', parse_dates=['timestamp'])

# Error summary by service and severity
error_summary = (
    logs[logs['level'].isin(['ERROR', 'CRITICAL'])]
    .groupby(['service', 'level'])
    .agg(
        count=('message', 'size'),
        first_seen=('timestamp', 'min'),
        last_seen=('timestamp', 'max')
    )
    .sort_values('count', ascending=False)
)

# Request latency percentiles by endpoint
latency_stats = (
    logs.groupby('endpoint')['response_time_ms']
    .agg([
        ('p50', lambda x: x.quantile(0.5)),
        ('p95', lambda x: x.quantile(0.95)),
        ('p99', lambda x: x.quantile(0.99))
    ])
)

print(error_summary.head(10))
print(latency_stats)

❌ Common Mistakes

1. Forgetting reset_index(): GroupBy returns Series with MultiIndex. Use .reset_index() to get a flat DataFrame.

2. Applying to wrong column: df.groupby('a').sum() sums ALL numeric columns. Be explicit: df.groupby('a')['b'].sum().

3. Using apply() when agg() works: .agg() is faster. Only use .apply() for complex custom functions.

✅ Quick Revision

  • df.groupby('col') — create GroupBy object
  • .sum(), .mean(), .count() — basic aggregations
  • .agg(['sum', 'mean']) — multiple aggregations
  • .agg(new_name=('col', 'func')) — named aggregations
  • .reset_index() — flatten back to DataFrame

23. Aggregation Functions

🧠 Mental Model

Aggregation is the act of collapsing data. You take a list of values (like a column or a group) and crush them down into a single number that summarizes them. Think of it as a funnel: many numbers go in, one number comes out (Sum, Average, Max).

💡 What Is It?

Functions that reduce the dimension of data. They take a Series (or Group) and return a single scalar value.

🔧 Why It Exists?

Raw data is too granular. We need summaries to understand trends. "What was total revenue?" is an aggregation question.

⚙️ How It Works?

Iterates over the array of values, performs a mathematical calculation (like summing), and returns the result. Ignores NaNs by default.

🎯 Where Used?

KPI reporting, dashboards, statistical analysis, feature engineering (creating summary features).

✅ When To Use?

When you need to summarize data. Getting counts, totals, averages, or spread (variance/std dev).

⚠️ When To Avoid?

When you need to maintain the original number of rows (use transform instead). When data is non-numeric (careful with max/min).

Simple Example

import pandas as pd
import numpy as np

df = pd.DataFrame({
    'A': [1, 2, 3, 4, 5],
    'B': [10, 20, 30, 40, 50]
})

# Single aggregation
print(df['A'].sum())   # 15
print(df['A'].mean())  # 3.0

# Describe entire dataframe
print(df.describe())

# Multiple aggregations
print(df.agg(['sum', 'min', 'max']))

# Specific aggregations per column
print(df.agg({
    'A': ['sum', 'mean'],
    'B': ['min', 'max']
}))

Production Example: Custom Agg

import pandas as pd

# Custom aggregation function
def percentile_90(x):
    return x.quantile(0.9)

def range_diff(x):
    return x.max() - x.min()

df = pd.DataFrame({
    'category': ['A', 'A', 'B', 'B'],
    'value': [10, 15, 10, 20]
})

# Applying custom functions via agg()
stats = df.groupby('category')['value'].agg([
    'mean',
    percentile_90,
    range_diff
])

print(stats)
#           mean  percentile_90  range_diff
# category
# A         12.5           14.5           5
# B         15.0           19.0          10

❌ Common Mistakes

1. Aggregating Strings: sum() on strings concatenates them ('a'+'b'='ab'). Use count() or nunique() for strings.

2. Ignoring NaNs: Pandas ignores NaNs by default. If you need strict math, check for NaNs first.

3. Confusing count() vs size(): count() counts non-null values. size() counts total rows (including NaNs).

✅ Quick Revision

  • .sum(), .mean(), .median() — Central tendency
  • .min(), .max(), .std(), .var() — Spread
  • .count() — Non-null count
  • .nunique() — Unique value count
  • .describe() — Summary stats
  • .agg() — Apply multiple functions

24. Transform vs Agg

🧠 Mental Model

Aggregation destroys rows (Many → One). Transform preserves rows (Many → Many). Think of transform as "broadcasting": it calculates a group statistic (like a group mean) and then pastes it back next to every original row in that group.

💡 What Is It?

Method that applies a function group-wise but returns an object indexed the same as original.

🔧 Why It Exists?

To combine group statistics with original data. E.g., "Is this student's score above the class average?"

⚙️ How It Works?

1. Splits into groups. 2. Applies function (e.g., mean). 3. Broadcasts result back to match original index.

🎯 Where Used?

Feature engineering (Group Mean Encoding), Outlier detection (Z-score by group), Imputing missing values with group median.

✅ When To Use?

When you want to add a "Group Stat" column to your original DataFrame without merging.

⚠️ When To Avoid?

When you want a summary table (use agg). When function changes row count (must return scalar or same-size array).

Simple Example

import pandas as pd

df = pd.DataFrame({
    'class': ['A', 'A', 'B', 'B'],
    'score': [80, 90, 60, 70]
})

# Calculate class average
class_avg = df.groupby('class')['score'].transform('mean')

# Add as new column
df['class_avg'] = class_avg
df['diff_from_avg'] = df['score'] - df['class_avg']

print(df)
#   class  score  class_avg  diff_from_avg
# 0     A     80       85.0           -5.0
# 1     A     90       85.0            5.0
# 2     B     60       65.0           -5.0
# 3     B     70       65.0            5.0

Production Example: Missing Values

import pandas as pd
import numpy as np

df = pd.DataFrame({
    'dept': ['IT', 'IT', 'HR', 'HR', 'IT'],
    'salary': [5000, np.nan, 4000, 4200, 5200]
})

# Fill missing salary with MEDIAN of that specific department
df['salary_filled'] = df['salary'].fillna(
    df.groupby('dept')['salary'].transform('median')
)

print(df)

# Filter out outliers (Z-score > 2 within group)
z_scores = df.groupby('dept')['salary_filled'].transform(
    lambda x: (x - x.mean()) / x.std()
)
clean_df = df[abs(z_scores) < 2]

❌ Common Mistakes

1. Using apply() instead of transform(): apply() is slower and might return a reduced result. Use transform() when you want to keep original shape.

2. Transforming entire DataFrame: Usually you want to transform a specific column. df.groupby('A').transform('mean') will try to average ALL columns.

3. Shuffling data: Transform preserves the index, so you can assign it back directly. Don't sort or shuffle before assigning.

✅ Quick Revision

  • .transform('mean') — Broadcast mean to rows
  • .transform(lambda x: ...) — Custom element-wise logic
  • fillna(groupby(...).transform(...)) — Smart imputation
  • Preserves index and length of original DataFrame

25. Window Functions

🧠 Mental Model

Window functions are like sliding glasses over your data. Instead of looking at the whole column or a fixed group, you look at a "window" of rows relative to the current row (e.g., "the previous 3 days"). Crucial for time-series: moving averages, cumulative sums, and trends.

💡 What Is It?

Operations that perform calculations across a set of rows that are related to the current row.

🔧 Why It Exists?

Data often has temporal dependence. Yesterday's sales affect today's context. We need to smooth noise or see cumulative progress.

⚙️ How It Works?

Defines a window (size N). Slides it one row at a time. Computes stat for that window. Assigns result to the current row.

🎯 Where Used?

Finance (Moving Averages), Sales (Year-to-Date), Sensor data (Smoothing), Lag features for ML.

✅ When To Use?

Calculating trends, smoothing, cumulative totals, percent change from previous period.

⚠️ When To Avoid?

When data isn't sorted (order matters!). When rows are independent observations.

Simple Example

import pandas as pd

df = pd.DataFrame({
    'date': pd.date_range('2024-01-01', periods=5),
    'sales': [10, 20, 30, 40, 50]
})

# Shift (Lag)
df['prev_day'] = df['sales'].shift(1)

# Rolling Window (Moving Average)
df['ma_3'] = df['sales'].rolling(window=3).mean()

# Expanding Window (Cumulative Sum)
df['cum_sales'] = df['sales'].expanding().sum()

print(df)
#         date  sales  prev_day  ma_3  cum_sales
# 0 2024-01-01     10       NaN   NaN       10.0
# 1 2024-01-02     20      10.0   NaN       30.0
# 2 2024-01-03     30      20.0  20.0       60.0
# 3 2024-01-04     40      30.0  30.0      100.0
# 4 2024-01-05     50      40.0  40.0      150.0

Production Example: Trends

import pandas as pd

# Financial data simulation
prices = pd.Series([100, 102, 101, 105, 110, 108, 107])

# Percent Change (Daily Returns)
returns = prices.pct_change()

# Exponential Moving Average (EMA)
# Gives more weight to recent data

# Rolling Z-Score (Anomaly Detection)
window = 3
roll_mean = prices.rolling(window).mean()
roll_std = prices.rolling(window).std()
z_score = (prices - roll_mean) / roll_std

print(f"EMA:\n{ema.values}")
print(f"Z-Score:\n{z_score.values}")

❌ Common Mistakes

1. Unsorted Data: Window functions assume order. If your data is shuffled, shift(1) gets a random row, not the previous day. Always sort_values() first!

2. Ignoring min_periods: By default, rolling window needs N values to compute. Use min_periods=1 if you want partial results at start.

3. Rolling vs Expanding: Rolling forgets old data (fixed size). Expanding remembers everything (growing size).

✅ Quick Revision

  • .shift(1) — Previous row value
  • .diff() — Difference from previous row
  • .pct_change() — % Growth
  • .rolling(window=N) — Fixed window
  • .expanding() — Cumulative window
  • .ewm() — Exponentially weighted functions

26. Resample Time Series

🧠 Mental Model

Think of Resampling as "GroupBy for Time". You split time into buckets (e.g., "Monthly", "Weekly", "5-Minute Intervals"), apply a function (Sum, Mean), and combine. Downsampling: High freq → Low freq (Minutes → Hours). Needs aggregation. Upsampling: Low freq → High freq (Days → Hours). Needs filling logic.

💡 What Is It?

Method to change the frequency of your time series data (e.g., Daily to Monthly).

🔧 Why It Exists?

To align data frequencies. Comparing daily sales vs monthly budget requires converting daily to monthly.

⚙️ How It Works?

Buckets the DateTimeIndex into defined intervals. Applies aggregation (for downsampling) or filling (for upsampling).

🎯 Where Used?

Financial charts (OHLC bars), IoT sensor aggregation (100Hz → 1Hz), aligning disparate datasets.

✅ When To Use?

Changing time granularity. "Convert 5-min data to Hourly average." "Fill missing weekend days."

⚠️ When To Avoid?

When your index is not a DateTimeIndex (it won't work). When you just want a rolling window (use rolling).

Simple Example

import pandas as pd
import numpy as np

# Create hourly data for 2 days
idx = pd.date_range('2024-01-01', periods=48, freq='H')
df = pd.DataFrame({'sales': np.random.randint(10, 100, 48)}, index=idx)

# Downsample: Hourly -> Daily Sum
daily_sales = df.resample('D').sum()

# Downsample: Hourly -> 6-Hour Mean
six_hour_avg = df.resample('6H').mean()

print(daily_sales)
print(six_hour_avg)

Production Example: OHLC & Upsampling

import pandas as pd

# Financial data (Tick data)
ticks = pd.DataFrame(
    {'price': [100, 102, 101, 105, 104]},
    index=pd.to_datetime([
        '09:30:00', '09:30:05', '09:31:00', '09:31:30', '09:32:00'
    ])
)

# Resample to 1-Minute OHLC (Open-High-Low-Close)
ohlc = ticks['price'].resample('1T').ohlc()
print(ohlc)

# Upsample: Minute -> Second (Forward Fill)
# Fills missing seconds with previous known value
seconds = ticks.resample('S').ffill()
print(seconds.head())

❌ Common Mistakes

1. No DateTimeIndex: resample() fails if index is not datetime. Use df.set_index('date_col') first.

2. Confusing asfreq() vs resample(): asfreq() selects values at specific times (no aggregation). resample() aggregates all values in the bin.

3. Upsampling NaNs: Upsampling creates new rows with NaNs. You must chain .ffill(), .bfill(), or .interpolate() to handle them.

✅ Quick Revision

  • .resample('D') / 'M' / 'Y' — Common frequencies
  • .resample('5T') — 5 Minutes
  • .sum(), .mean(), .max() — Aggregation functions
  • .ohlc() — Open-High-Low-Close bars
  • .ffill() / .interpolate() — Filling logic for upsampling

27. DateTime Basics

🧠 Mental Model

Pandas has a dedicated Timestamp object. It's like Python's native datetime but supercharged. A column of these is the foundation for all time-series magic. Rule #1: Never leave dates as strings ("2024-01-01"). Always convert to Timestamp.

💡 What Is It?

Data type for storing specific points in time (YYYY-MM-DD HH:MM:SS). Stored as 64-bit integers (nanoseconds since epoch).

🔧 Why It Exists?

Strings are dumb; Timestamps are smart. Timestamps know the day of the week, quarter, leap years, and time intervals.

⚙️ How It Works?

Uses pd.to_datetime() to parse strings. Exposes the .dt accessor for extracting components (year, month, hour).

🎯 Where Used?

Any dataset with dates: Sales transactions, Server logs, User signups, Stock prices.

✅ When To Use?

Immediately upon loading data. "Convert 'date' column to datetime." "Filter for weekends."

⚠️ When To Avoid?

When you only need the year as a simple integer. But usually, Timestamp is still better.

Simple Example

import pandas as pd

df = pd.DataFrame({
    'date_str': ['2024-01-01', '2024-02-15', '2024-03-30'],
    'value': [10, 20, 30]
})

# Convert to datetime
df['date'] = pd.to_datetime(df['date_str'])

# Extract components via .dt accessor
df['year'] = df['date'].dt.year
df['month'] = df['date'].dt.month_name()
df['day_of_week'] = df['date'].dt.day_name()

print(df)

Production Example: Parsing & Errors

import pandas as pd

# Messy data
dates = ['2024-01-01', '01/02/2024', 'Not a Date', '2024-02-30']

# Coerce errors: turns invalid dates into NaT (Not a Time)

28. Timedelta

🧠 Mental Model

If Timestamp is a specific point on the timeline (Jan 1st), Timedelta is the distance between two points (3 days, 5 hours). It's the result of Time B - Time A. You can add Timedeltas to Timestamps to move forward in time.

💡 What Is It?

Object representing a duration or difference between two times. Like Python's datetime.timedelta.

🔧 Why It Exists?

To measure elapsed time. "How long did the user stay on the page?" "Time to resolve ticket."

⚙️ How It Works?

Created by subtracting two Timestamps or manually via pd.Timedelta(). Supports arithmetic (Date + Timedelta = New Date).

🎯 Where Used?

Session duration analysis, SLA tracking, Age calculation, Cohort analysis (Days since signup).

✅ When To Use?

Calculating time differences. Adding fixed durations ("Expiry is 30 days from now").

⚠️ When To Avoid?

Using integers for time (e.g., storing "5" for 5 days). Always use Timedelta for clarity.

Simple Example

import pandas as pd

# Create Timedelta manually
delta = pd.Timedelta('2 days 5 hours')
print(delta)

# Date Arithmetic
start = pd.Timestamp('2024-01-01')
end = start + delta
print(end) # 2024-01-03 05:00:00

# Subtracting dates creates Timedelta
diff = end - start
print(diff) # 2 days 05:00:00

Production Example: Session Duration

import pandas as pd

df = pd.DataFrame({
    'login': pd.to_datetime(['2024-01-01 09:00', '2024-01-01 10:00']),
    'logout': pd.to_datetime(['2024-01-01 09:30', '2024-01-01 10:05'])
})

# Calculate duration
df['duration'] = df['logout'] - df['login']

# Convert to minutes for analysis
df['minutes'] = df['duration'].dt.total_seconds() / 60

# Filter sessions > 10 mins
long_sessions = df[df['duration'] > pd.Timedelta('10 min')]

print(df)

❌ Common Mistakes

1. Using .dt.seconds vs .dt.total_seconds(): .seconds only returns the seconds part (ignoring days!). Always use .total_seconds() for full duration.

2. Inconsistent Units: Adding integer 5 to a date is ambiguous. Is it 5 days? 5 nanoseconds? Use pd.Timedelta.

3. Negative Durations: If End Time < Start Time, you get negative Timedelta. Check your data quality.

✅ Quick Revision

  • pd.Timedelta('1 days') — Create duration
  • SeriesA - SeriesB — Result is Timedelta
  • .dt.days — Extract days component
  • .dt.total_seconds() — Convert to seconds (float)
  • pd.to_timedelta() — Convert strings/numbers to Timedelta

29. DateTime Index

🧠 Mental Model

When you move a Timestamp column into the Index, it becomes a DatetimeIndex. This unlocks superpowers: Partial String Indexing (slice by "2024" or "Jan 2024"), automatic alignment, and Resampling. It turns a standard table into a Time Series.

💡 What Is It?

An Index composed entirely of Timestamp objects. Optimized for time-based lookups.

🔧 Why It Exists?

To make time-based filtering intuitive and fast. "Give me all data from January" becomes one line of code.

⚙️ How It Works?

Use set_index('date_col'). Ensure it is sorted. Pandas then uses specialized algorithms for slicing.

🎯 Where Used?

Financial analysis, Log analysis, any dataset where "Time" is the primary dimension.

✅ When To Use?

Always, if your analysis revolves around time. Essential for resample and rolling.

If time is just one of many attributes (e.g., "User Birthdate") and not the primary axis of analysis.

30. Rolling Windows

🧠 Mental Model

Think of a "Moving Spotlight". You look at 3 days of data, calculate the average, then slide the spotlight one day to the right and repeat. This smooths out noise in time series data (like stock prices or temperature).

💡 What Is It?

Calculations performed on a sliding window of fixed size (e.g., "7-day moving average").

🔧 Why It Exists?

To smooth short-term fluctuations and highlight longer-term trends.

⚙️ How It Works?

df.rolling(window=N).mean(). The result is aligned to the right edge of the window.

🎯 Where Used?

Finance (SMA/EMA), Signal Processing, Sales Forecasting.

✅ When To Use?

When analyzing trends over time. "Is the trend going up despite daily noise?"

⚠️ When To Avoid?

If data is not sorted by time! (Rolling assumes order). If you have large gaps in time (use rolling('7D') instead of window=7).

Simple Moving Average

import pandas as pd
import numpy as np

# Daily stock price
df = pd.DataFrame({'price': [10, 12, 14, 13, 15]})

# 3-Day Rolling Mean
# Day 1-2 will be NaN (not enough data)
df['sma_3'] = df['price'].rolling(3).mean()

print(df)
#    price  sma_3
# 0     10    NaN
# 1     12    NaN
# 2     14   12.0  ( (10+12+14)/3 )
# 3     13   13.0
# 4     15   14.0

Production: Time-Based Window

# If you have missing days, use '3D' (3 Days)
# instead of window=3 (3 Rows)
df = df.set_index(pd.to_datetime([
    '2024-01-01', '2024-01-02', '2024-01-05', 
    '2024-01-06', '2024-01-07'
]))

# Rolling 3-Day Sum
res = df.rolling('3D').sum()

print(res)

❌ Common Mistakes

1. Center Alignment: By default, the label is at the end of the window. Use center=True to put the label in the middle (lookahead bias!).

2. Rows vs Time: window=7 looks at 7 rows. window='7D' looks at 7 days. They are different if you have missing weekends.

3. Expanding: If you want "Cumulative Average" (from start to now), use df.expanding().mean() instead of rolling.

✅ Quick Revision

  • .rolling(3).mean() — Simple Moving Average
  • .rolling('7D').sum() — Time-aware rolling
  • .expanding().sum() — Cumulative Sum (Running Total)
  • .rolling(3, min_periods=1) — Avoid NaNs at start
  • .diff() — Discrete difference (not rolling, but related)

31. Time Zones

🧠 Mental Model

By default, Timestamps are Naive (clueless about location). Aware Timestamps know their offset from UTC. Think of Naive as "Wall Clock Time" (it's 5 PM here). Think of Aware as "Absolute Time" (it's 5 PM UTC-5). Golden Rule: Always convert everything to UTC for processing. Only convert to local time for display.

💡 What Is It?

Handling different time offsets (UTC, EST, CET). Uses pytz or dateutil under the hood.

🔧 Why It Exists?

To compare events across the globe. "Did the Tokyo market open before NY closed?" "Syncing logs from US and EU servers."

⚙️ How It Works?

Two steps: 1. tz_localize() (Naive → Aware). 2. tz_convert() (Aware → Aware).

🎯 Where Used?

Global applications, Server logs (always UTC), Financial markets, Flight schedules.

✅ When To Use?

When your data comes from different regions. When you need to handle Daylight Savings Time (DST).

⚠️ When To Avoid?

Mixing Naive and Aware timestamps in the same column (Pandas will convert to object dtype). Pick one!

Simple Example

import pandas as pd

# Naive Timestamp
ts = pd.Timestamp('2024-01-01 12:00')
print(ts)

# Localize to UTC (Make it Aware)
ts_utc = ts.tz_localize('UTC')
print(ts_utc) # 2024-01-01 12:00:00+00:00

# Convert to NY Time
ts_ny = ts_utc.tz_convert('US/Eastern')
print(ts_ny) # 2024-01-01 07:00:00-05:00

Production Example: Normalizing Logs

import pandas as pd

df = pd.DataFrame({
    'server': ['US', 'EU'],
    'local_time': ['2024-01-01 10:00', '2024-01-01 10:00']
})

# Convert to datetime first
df['local_time'] = pd.to_datetime(df['local_time'])

# Localize individually (since offsets differ)
df.loc[df['server'] == 'US', 'utc_time'] = (
    df.loc[df['server'] == 'US', 'local_time']
    .dt.tz_localize('US/Eastern')
    .dt.tz_convert('UTC')
)

df.loc[df['server'] == 'EU', 'utc_time'] = (
    df.loc[df['server'] == 'EU', 'local_time']
    .dt.tz_localize('Europe/Berlin')
    .dt.tz_convert('UTC')
)

print(df[['server', 'utc_time']])

❌ Common Mistakes

1. Assuming Naive is UTC: If you tz_convert a naive timestamp, Pandas errors. You MUST tz_localize first.

2. Daylight Savings Time (DST): Naive math fails around DST changes. Aware math handles it automatically.

3. Storing Local Time: Always store UTC in databases/files. Only convert to local time at the very last step (visualization).

✅ Quick Revision

  • .dt.tz_localize('UTC') — Set initial timezone
  • .dt.tz_convert('US/Eastern') — Change timezone
  • .dt.tz_localize(None) — Remove timezone (make naive)
  • pd.Timestamp.now('UTC') — Current UTC time

32. Vectorization

🧠 Mental Model

Vectorization is Batch Processing. Instead of looping through rows one by one (like a Python for loop), you apply an operation to the entire column at once. Under the hood, Pandas uses optimized C code (NumPy) to process the array instantly. It's the difference between a single worker and an industrial assembly line.

💡 What Is It?

Performing operations on entire arrays/columns simultaneously instead of element-by-element iteration.

🔧 Why It Exists?

Speed. Python loops are slow due to overhead. Vectorized operations run in compiled C, often 100x-1000x faster.

⚙️ How It Works?

Use built-in Pandas/NumPy methods (`+`, `*`, `.str`, `.dt`). Avoid explicit loops.

🎯 Where Used?

Everywhere. Math calculations, string manipulation, date logic, conditional logic.

✅ When To Use?

Always. It should be your default. Only drop to loops/apply if no vectorized alternative exists.

⚠️ When To Avoid?

iterrows(), itertuples(), and apply() (unless logic is too complex for vectorization).

Slow (Loop/Apply)

import pandas as pd
import time

df = pd.DataFrame({'a': range(100000), 'b': range(100000)})

# BAD: Iterrows (Extremely Slow)
start = time.time()
res = []
for idx, row in df.iterrows():
    res.append(row['a'] + row['b'])
print(f"Loop: {time.time()-start:.4f}s")

# BAD: Apply (Hidden Loop)
start = time.time()
res = df.apply(lambda x: x['a'] + x['b'], axis=1)
print(f"Apply: {time.time()-start:.4f}s")

Fast (Vectorized)

import pandas as pd
import numpy as np

# GOOD: Vectorized Math (Instant)
start = time.time()
res = df['a'] + df['b']
print(f"Vector: {time.time()-start:.4f}s")

# GOOD: Vectorized Conditional (np.where)
# If a > b then 'High' else 'Low'
df['status'] = np.where(df['a'] > df['b'], 'High', 'Low')

# GOOD: Vectorized String Ops
df['text'] = 'Item ' + df['a'].astype(str)

❌ Common Mistakes

1. Thinking apply() is fast: It's not. It loops in Python. It's convenient, but slow.

2. Using List Comprehensions unnecessarily: List comprehensions are faster than loops but slower than vectorization for large data.

3. Ignoring NumPy: np.where, np.select, and np.log are your friends for complex vectorized logic.

✅ Quick Revision

  • df['a'] + df['b'] — Basic Math
  • .str.upper() — String Vectorization
  • .dt.year — Date Vectorization
  • np.where(cond, x, y) — Vectorized If-Else
  • np.select(conds, choices) — Vectorized Elif

33. Memory Optimization

🧠 Mental Model

Pandas is RAM-hungry. By default, it uses huge containers (int64, float64, object) even for tiny numbers. Optimization involves "Downcasting" (shrinking the container to fit the data, e.g., int8) and "Categorical Encoding" (replacing repeated strings with lightweight integers).

💡 What Is It?

Techniques to drastically reduce the RAM usage of a DataFrame (often by 50-90%).

🔧 Why It Exists?

To fit larger datasets into memory. "My 4GB CSV crashes my 8GB laptop."

⚙️ How It Works?

Changing dtypes: int64 → int16, float64 → float32, object → category.

🎯 Where Used?

Big Data processing, Cloud functions (AWS Lambda with RAM limits), Local development.

✅ When To Use?

When you hit MemoryError. When operations become sluggish due to swapping.

⚠️ When To Avoid?

Premature optimization on small datasets (< 100MB). If it fits easily, don't bother.

Standard (Heavy)

import pandas as pd
import numpy as np

# Default dtypes are large
df = pd.DataFrame({
    'age': np.random.randint(0, 100, 100000), # int64
    'score': np.random.rand(100000),          # float64
    'grade': ['A'] * 100000                   # object (string)
})

# Check memory usage (deep=True for strings)
print(df.info(memory_usage='deep'))
# Memory usage: ~6.5 MB

Optimized (Light)

# Downcast Numbers
df['age'] = df['age'].astype('int8')    # 0-100 fits in int8
df['score'] = df['score'].astype('float32')

# Convert Strings to Category
df['grade'] = df['grade'].astype('category')

print(df.info(memory_usage='deep'))
# Memory usage: ~0.6 MB (90% reduction!)

❌ Common Mistakes

1. Downcasting too much: Converting int64 to int8 when values exceed 127 causes overflow (garbage data). Check .max() first.

2. Categorizing unique strings: category dtype is only useful if strings repeat often (low cardinality). If every string is unique, it uses more memory.

3. Not using 'deep': df.memory_usage() ignores string memory unless you pass deep=True.

✅ Quick Revision

  • df.info(memory_usage='deep') — Check real size
  • .astype('int8/16/32') — Smaller integers
  • .astype('float32') — Smaller floats
  • .astype('category') — For repeated strings
  • pd.to_numeric(downcast='integer') — Auto-downcast

34. Category Dtype

🧠 Mental Model

Category is a dictionary-encoded type. Instead of storing "Apple", "Apple", "Apple" (3 heavy strings), Pandas stores "Apple" once in a lookup table and stores 0, 0, 0 (lightweight integers) in the column. Think of it as a Lookup Table or Enum.

💡 What Is It?

A special Pandas dtype for categorical data (finite list of text values).

🔧 Why It Exists?

1. Memory: Saves RAM if values repeat. 2. Speed: Grouping/Sorting by integer codes is faster than strings.

⚙️ How It Works?

Separates data into codes (integers) and categories (unique labels). Use .astype('category').

🎯 Where Used?

Survey responses (Yes/No), City names, Product categories, Grades (A/B/C), Days of Week.

✅ When To Use?

When a string column has few unique values (low cardinality) relative to total rows (Ratio < 50%).

⚠️ When To Avoid?

High-cardinality data (User IDs, GUIDs, unique names). It will use more memory and be slower.

Simple Example

import pandas as pd

df = pd.DataFrame({
    'city': ['NY', 'LA', 'NY', 'Chicago', 'LA']
})

# Convert to Category
df['city_cat'] = df['city'].astype('category')

print(df['city_cat'])
# Categories (3, object): ['Chicago', 'LA', 'NY']

# View underlying codes
print(df['city_cat'].cat.codes)
# [2, 1, 2, 0, 1] (Integers!)

Production Example: Ordered Categories

import pandas as pd

df = pd.DataFrame({
    'size': ['M', 'S', 'L', 'S', 'XL']
})

# Define logical order
sizes = ['S', 'M', 'L', 'XL']

# Create Ordered Category
df['size_cat'] = pd.Categorical(
    df['size'],
    categories=sizes,
    ordered=True
)

# Now we can sort logically (S < M < L)
print(df.sort_values('size_cat'))

# Filter: Sizes larger than M
print(df[df['size_cat'] > 'M'])

❌ Common Mistakes

1. High Cardinality: Converting unique IDs to category is a disaster. It creates a huge lookup table.

2. Adding new values: You can't set df['cat'][0] = 'New' if 'New' isn't in the categories. Use .cat.add_categories('New') first.

3. Merging on Categories: If categories don't match exactly between two DataFrames, merges can be tricky or slow. Sometimes safer to convert back to string for merge.

✅ Quick Revision

  • .astype('category') — Basic conversion
  • pd.Categorical(ordered=True) — For logical sorting
  • .cat.codes — Access integer codes
  • .cat.categories — Access unique labels
  • .cat.add_categories() — Modify allowed values

35. Chunked Processing

🧠 Mental Model

When a file is too big for RAM (e.g., 10GB file on 8GB laptop), you can't read it all at once. Instead, you read it in Chunks (small bites), process each chunk (filter/aggregate), and combine the results. Think of it as Streaming data.

💡 What Is It?

Reading a large file in smaller, manageable pieces (e.g., 10,000 rows at a time).

🔧 Why It Exists?

To process datasets larger than your machine's physical memory (Out-of-Core processing).

⚙️ How It Works?

Use pd.read_csv(..., chunksize=N). It returns an iterator, not a DataFrame.

🎯 Where Used?

ETL pipelines, Log analysis, processing huge historical datasets.

✅ When To Use?

File size > Available RAM. Or when you only need to filter a small subset of a huge file.

⚠️ When To Avoid?

If the file fits in memory, just read it normally. Chunking adds code complexity and overhead.

Simple Example

import pandas as pd

# Create a dummy large file
# (In reality, this would be a real file)

# Read in chunks of 100 rows
reader = pd.read_csv('large_file.csv', chunksize=100)

for chunk in reader:
    # chunk is a DataFrame of 100 rows
    print(chunk.shape)
    # Process chunk here...

Production Example: Aggregating Chunks

import pandas as pd

chunk_size = 10000
total_sales = 0
filtered_chunks = []

with pd.read_csv('sales_data.csv', chunksize=chunk_size) as reader:
    for chunk in reader:
        # 1. Aggregate: Add to running total
        total_sales += chunk['amount'].sum()
        
        # 2. Filter: Keep only high-value rows
        high_value = chunk[chunk['amount'] > 1000]
        filtered_chunks.append(high_value)

# Combine filtered results
final_df = pd.concat(filtered_chunks)

print(f"Total Sales: {total_sales}")
print(final_df.head())

❌ Common Mistakes

1. Converting iterator to list: list(reader) loads ALL chunks into memory at once, crashing your machine. Process one by one!

2. Averaging Averages: You can't calculate the mean by averaging the means of chunks. You must sum the totals and sum the counts, then divide at the end.

3. Chunksize too small: Reading 1 row at a time is incredibly slow. Use 10k-100k depending on row width.

✅ Quick Revision

  • pd.read_csv(chunksize=N) — Returns TextFileReader
  • for chunk in reader: — Iterate through chunks
  • pd.concat(results) — Combine filtered chunks
  • chunk.to_csv(mode='a') — Append chunk to file

36. Numba Integration

🧠 Mental Model

When even Vectorization isn't fast enough, or you have complex logic that cannot be vectorized (e.g., iterative calculations where row N depends on row N-1), Numba compiles your Python function into machine code (JIT - Just In Time). Think of it as "Turbo Mode" for custom functions.

💡 What Is It?

A JIT compiler that translates Python functions to optimized machine code (LLVM).

🔧 Why It Exists?

To speed up custom apply() functions or loops that cannot be vectorized. 100x speedups are common.

⚙️ How It Works?

Decorate your function with @numba.jit. Pandas also supports engine='numba' in some methods.

🎯 Where Used?

Financial modeling (path-dependent options pricing), complex simulations, custom rolling windows.

✅ When To Use?

When you have a slow loop/apply that is the bottleneck, and vectorization is impossible.

⚠️ When To Avoid?

Simple tasks where built-in Pandas methods already exist. Compilation overhead makes it slower for small data.

Standard (Slow Apply)

import pandas as pd
import numpy as np

df = pd.DataFrame({'a': np.random.randn(100000)})

def slow_func(x):
    res = 0
    for val in x:
        if val > 0:
            res += val
    return res

# Standard Rolling Apply
# Very slow because it calls Python function 100k times
res = df['a'].rolling(100).apply(slow_func)

Numba (Fast JIT)

from numba import jit

# JIT Compile the function
# nopython=True ensures it runs in machine code
@jit(nopython=True)
def fast_func(values):
    res = 0
    for val in values:
        if val > 0:
            res += val
    return res

# Use engine='numba' and raw=True
# raw=True passes numpy arrays instead of Series (faster)
res = df['a'].rolling(100).apply(
    fast_func, 
    engine='numba', 
    raw=True
)

❌ Common Mistakes

1. Not using nopython=True: Without this (or @njit), Numba might fall back to slow Python mode if it can't compile something.

2. Using unsupported features: Numba doesn't support all Python features (e.g., dictionaries inside the function). Keep it to simple math and loops.

3. Compilation Overhead: The first time you run a JIT function, it takes time to compile. Don't use it for one-off small tasks.

✅ Quick Revision

  • @jit(nopython=True) — The decorator to use
  • engine='numba' — Enable in Pandas methods
  • raw=True — Pass NumPy arrays (Critical for speed)
  • from numba import prange — Parallel loops

37. MultiIndex

🧠 Mental Model

A MultiIndex (Hierarchical Index) is like a "Pivot Table" structure but in the Index. It allows you to group data by multiple levels (e.g., "State" → "City") without losing the tabular structure. Think of it as "Sub-folders" for your rows.

💡 What Is It?

An Index with multiple levels (e.g., Level 0: Year, Level 1: Month).

🔧 Why It Exists?

To represent higher-dimensional data (3D/4D) in a 2D DataFrame. "Sales by Region AND Product."

⚙️ How It Works?

Created via set_index(['col1', 'col2']) or groupby(['col1', 'col2']).

🎯 Where Used?

Pivot tables, Time series (Stock Ticker + Date), Grouped aggregations.

✅ When To Use?

When you need to slice by multiple dimensions ("Give me all data for 'NY' in '2024'").

⚠️ When To Avoid?

If you don't need it. It adds complexity. Often, resetting the index (reset_index()) to a flat table is easier.

Simple Example

import pandas as pd

df = pd.DataFrame({
    'state': ['NY', 'NY', 'CA', 'CA'],
    'city': ['NYC', 'Albany', 'SF', 'LA'],
    'sales': [100, 50, 80, 90]
})

# Create MultiIndex
df = df.set_index(['state', 'city'])
df = df.sort_index() # Important!

print(df)
#               sales
# state city         
# CA    LA         90
#       SF         80
# NY    Albany     50
#       NYC       100

# Slice Outer Level (State)
print(df.loc['NY'])

# Slice Specific Row (Tuple)
print(df.loc[('NY', 'NYC')])

Production Example: Cross-Section (xs)

import pandas as pd

# Select Inner Level across all Outer Levels
# e.g., "Get 'NYC' data from ALL states" (if NYC existed in multiple states)

# Using .xs (Cross Section)
# level=1 refers to 'city'
nyc_data = df.xs('NYC', level='city')

print(nyc_data)
#        sales
# state       
# NY       100

# Reset to flat table (often easier for export)
flat_df = df.reset_index()

❌ Common Mistakes

1. Unsorted Index: Slicing a MultiIndex fails or is slow if it's not sorted. Always call sort_index() after creating it.

2. Syntax Confusion: df.loc['NY', 'NYC'] works, but df.loc[('NY', 'NYC')] is more explicit. Be careful with brackets.

3. Complexity Overload: If you find yourself fighting the MultiIndex, just reset_index() and use boolean filtering. It's often simpler.

✅ Quick Revision

  • set_index(['a', 'b']) — Create MultiIndex
  • sort_index() — Mandatory for slicing
  • loc[('a', 'b')] — Select specific row
  • xs('val', level=1) — Select inner level
  • reset_index() — Flatten back to columns

39. Query Method

🧠 Mental Model

df.query() is like writing SQL inside Python. Instead of the verbose df[df['col'] > 5], you write df.query('col > 5'). It's cleaner, easier to read, and often faster for large datasets (uses NumExpr). Think of it as "Natural Language Filtering".

💡 What Is It?

A string-based method to filter DataFrames using boolean expressions.

🔧 Why It Exists?

Readability. It removes the clutter of repeated df[...] brackets and parentheses.

⚙️ How It Works?

df.query('Condition'). Use @variable to reference external Python variables.

🎯 Where Used?

Complex filtering logic, interactive dashboards, data exploration.

✅ When To Use?

When you have multiple conditions (A > 5 & B < 10) and want clean code.

⚠️ When To Avoid?

Simple filters where standard indexing is fine. Or if column names contain weird characters (though backticks help).

Standard (Verbose)

import pandas as pd

df = pd.DataFrame({
    'A': [1, 5, 10],
    'B': [2, 4, 6],
    'City': ['NY', 'LA', 'NY']
})

limit = 4

# Standard Boolean Indexing
# Hard to read with many brackets
res = df[
    (df['A'] > df['B']) & 
    (df['B'] >= limit)
]

print(res)

Query (Clean)

# Using Query Method
# 1. Column names used directly (A, B)
# 2. @ references external variable
res = df.query('A > B and B >= @limit')

print(res)

# String filtering
# Note the quotes: "NY" inside '...'
ny_df = df.query('City == "NY"')

❌ Common Mistakes

1. Quote Confusion: query('City == NY') fails because it thinks NY is a column. Use query('City == "NY"').

2. Spaces in Columns: If a column is "Sales Amount", you must use backticks: query('`Sales Amount` > 100').

3. Assignment: query returns a new DataFrame. It doesn't modify in-place. Use df = df.query(...).

✅ Quick Revision

  • .query('A > B') — Basic comparison
  • .query('A > @val') — Use Python variable
  • .query('`Col Name` > 0') — Handle spaces
  • .query('City in ["NY", "LA"]') — List membership
  • .query('not C') — Boolean negation

40. Method Chaining

🧠 Mental Model

Method Chaining is the Pipeline approach. Instead of creating intermediate variables (df1, df2, df3), you chain operations together (df.filter().groupby().mean()). It reads like a recipe: "Take data, THEN filter, THEN group, THEN average."

💡 What Is It?

Calling multiple methods sequentially on the same object in one statement.

🔧 Why It Exists?

Cleaner code, no temporary variables, easier to read (if formatted correctly).

⚙️ How It Works?

df.method1().method2(). Wrap in parentheses (...) to allow multi-line formatting.

🎯 Where Used?

Data cleaning pipelines, ETL scripts, modern Pandas codebases.

✅ When To Use?

When you have a linear sequence of transformations.

⚠️ When To Avoid?

Debugging long chains can be hard. If a chain gets too complex (10+ steps), break it up.

Traditional (Messy)

import pandas as pd

df = pd.read_csv('data.csv')

# Intermediate variables clutter namespace
df1 = df.dropna()
df2 = df1[df1['sales'] > 100]
df3 = df2.groupby('city').sum()
final_df = df3.sort_values('sales')

print(final_df)

Chained (Clean)

# The "Fluent" Interface
# Wrap in () for multi-line support
final_df = (
    pd.read_csv('data.csv')
    .dropna()
    .query('sales > 100')
    .groupby('city')
    .sum()
    .sort_values('sales')
)

print(final_df)

❌ Common Mistakes

1. Missing Parentheses: Python requires \ for line breaks unless the code is inside (). Always wrap chains in parentheses.

2. In-place operations: Methods like inplace=True return None, breaking the chain. Never use inplace=True in a chain.

3. Debugging: "Where did it go wrong?" Use .pipe() to insert a print statement in the middle of a chain for debugging.

✅ Quick Revision

  • ( ... ) — Wrap the whole chain
  • .assign(new_col=...) — Create columns in chain
  • .query('...') — Filter in chain
  • .pipe(func) — Apply custom function
  • .rename(columns=...) — Rename in chain

41. Extension Types

🧠 Mental Model

Pandas originally relied on NumPy for data types (int64, float64, object). But NumPy lacks native support for things like "Nullable Integers" (NaN in an int column forces it to float) or "Strings" (stored as generic objects). Extension Types are Pandas' modern way to fix this. Think of them as "Plugins" that upgrade standard NumPy types.

💡 What Is It?

Custom data types defined by Pandas (not NumPy) to handle missing data and specific formats better.

🔧 Why It Exists?

To allow NaN in integer columns (Int64), faster string processing (StringDtype), and boolean with NaNs.

⚙️ How It Works?

Use dtype="Int64" (Capital I), dtype="string", dtype="boolean".

🎯 Where Used?

Modern Pandas code (Pandas 1.0+), especially when dealing with missing data.

✅ When To Use?

When you need to preserve integer precision but have missing values. Or want safer string operations.

⚠️ When To Avoid?

Legacy codebases might not expect them (some older libraries might crash). Check compatibility.

Old Way (NumPy)

import pandas as pd
import numpy as np

# Problem 1: NaN forces Int to Float
s_int = pd.Series([1, 2, np.nan])
print(s_int)
# 0    1.0
# 1    2.0
# 2    NaN
# dtype: float64 (Precision lost!)

# Problem 2: Strings are Objects
s_str = pd.Series(['a', 'b', np.nan])
print(s_str.dtype)
# dtype: object (Slow, mixed types allowed)

New Way (Extension)

# Solution 1: Nullable Integer (Int64)
# Note the Capital 'I'
s_int = pd.Series([1, 2, np.nan], dtype="Int64")
print(s_int)
# 0       1
# 1       2
# 2    <NA>
# dtype: Int64 (Integers preserved!)

# Solution 2: StringDtype
s_str = pd.Series(['a', 'b', np.nan], dtype="string")
print(s_str.dtype)
# dtype: string (Enforces text only)

❌ Common Mistakes

1. int64 vs Int64: int64 (lowercase) is NumPy (fast, no NaN). Int64 (Capital) is Pandas (supports NaN). Don't mix them up.

2. Object vs String: object can hold anything (ints, lists, strings). string dtype guarantees text. Always prefer string for text data.

3. convert_dtypes(): Use df.convert_dtypes() to automatically upgrade standard types to these new Extension Types.

✅ Quick Revision

  • dtype="Int64" — Nullable Integer
  • dtype="string" — Dedicated String type
  • dtype="boolean" — Nullable Boolean
  • df.convert_dtypes() — Auto-convert best types
  • pd.NA — The new missing value indicator

42. Serialization

🧠 Mental Model

Serialization is "Saving your game". You have a DataFrame in RAM (the game state), and you need to write it to disk so you can load it later. Different formats have different pros/cons (CSV is human-readable but slow; Parquet is binary, compressed, and fast; Pickle is Python-specific and fragile).

💡 What Is It?

Saving DataFrames to files (and reading them back).

🔧 Why It Exists?

Persistence, sharing data, checkpointing long processes.

⚙️ How It Works?

to_csv(), to_parquet(), to_pickle(), to_json().

🎯 Where Used?

Data pipelines, saving model training data, archiving.

✅ When To Use?

At the end of a script, or as intermediate steps in a pipeline.

⚠️ When To Avoid?

to_csv for huge data (slow, loses types). to_pickle for long-term storage (unsafe, version issues).

CSV (Human Readable)

import pandas as pd

df = pd.DataFrame({'A': [1, 2], 'B': ['x', 'y']})

# Save to CSV
# index=False prevents saving the row numbers
df.to_csv('data.csv', index=False)

# Read back
df2 = pd.read_csv('data.csv')

# Pros: Can open in Excel, text editor.
# Cons: Slow, large file size, loses dtypes.

Parquet (Fast & Efficient)

# Save to Parquet
# Requires 'pyarrow' or 'fastparquet' library
df.to_parquet(
    'data.parquet', 
    compression='snappy'
)

# Read back
df3 = pd.read_parquet('data.parquet')

# Pros: Very fast, small (compressed), keeps dtypes.
# Cons: Binary format (can't open in Notepad).

❌ Common Mistakes

1. Forgetting index=False: In CSV, this creates an annoying "Unnamed: 0" column when you read it back. Always use index=False unless the index is meaningful.

2. Using Pickle for Sharing: Pickle is insecure (can execute code) and breaks between Python versions. Only use it for temporary local storage.

3. Not using compression: For large files, compression='gzip' or 'snappy' saves huge disk space.

✅ Quick Revision

  • to_csv(index=False) — Standard text format
  • to_parquet() — Best for Big Data (Fast/Small)
  • to_json() — For web APIs
  • to_excel() — For business reports
  • to_pickle() — Temporary Python-only storage
